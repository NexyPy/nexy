# مرجع CLI

تعد واجهة سطر أوامر Nexy (`nx`) أداة قوية لإدارة تدفق التطوير الخاص بك.

## الأوامر

### `nx init`

يهيئ مشروع Nexy جديد. اتبع المساعد التفاعلي لاختيار إطار عمل الواجهة الأمامية الخاص بك، وتفعيل Tailwind CSS، وتكوين هيكل مشروعك.

```bash
nx init my-app
```

### `nx dev`

يبدأ خادم تطوير Nexy. يبدأ هذا الأمر واجهة FastAPI الخلفية ومنسق Vite بالتوازي، مع تفعيل التبديل الساخن للوحدات (HMR).

```bash
nx dev
```

### `nx build`

يجمع أصولك ويجهز مشروعك للنشر في الإنتاج. يعمل هذا الأمر على تحسين الكود والأصول الخاصة بك للحصول على أداء أفضل.

```bash
nx build
```

### `nx start`

يبدأ خادم إنتاج Nexy. ينفذ هذا الأمر تطبيقك في وضع الإنتاج دون الأعباء الإضافية لخادم التطوير.

```bash
nx start
```

### `nx add`

يضيف مكونات أو ميزات جديدة إلى مشروع Nexy الخاص بك.

```bash
nx add auth
```

### `nx deploy`

ينشر تطبيق Nexy الخاص بك لدى مزود سحابي.

```bash
nx deploy
```

---

**نصيحة**: استخدم `nx --help` للحصول على قائمة كاملة بالأوامر والخيارات.
