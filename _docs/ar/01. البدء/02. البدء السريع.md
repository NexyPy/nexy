# البدء السريع: من API إلى Fullstack في 5 دقائق

تم تصميم هذا الدليل للمطورين الذين يرغبون في فهم مرونة Nexy بسرعة، بدءًا من واجهة خلفية قوية وانتهاءً بواجهة مختلطة.

## 1. التثبيت

```bash
pip install nexy
```

## 2. تهيئة مشروع "Pure Backend"

لنبدأ بإنشاء مشروع بدون واجهة أمامية في الوقت الحالي.

```bash
nx init my-project --no-frontend
cd my-project
```

## 3. إنشاء واجهة برمجة تطبيقات (API) معيارية

افتح `src/app/main.py` وقم بتعريف متحكم بسيط:

```python
from nexy.decorators import Controller

@Controller("/api")
class HelloController:
    # يحدد اسم الطريقة فعل HTTP (get، post، إلخ.)
    async def get(self):
        return {"status": "Nexy is running", "mode": "Pure Backend"}
```

ابدأ الخادم:
```bash
nx dev
```
قم بزيارة `http://localhost:8000/api`. لديك الآن واجهة خلفية FastAPI سريعة للغاية ومعيارية.

## 4. الانتقال إلى Fullstack (تأثير "Wow")

الآن، لنقم بإضافة واجهة مستخدم دون تعديل الواجهة الخلفية الخاصة بنا. ما عليك سوى إنشاء ملف `src/routes/index.nexy`:

```html
---
# منطق بايثون مدمج
from datetime import datetime
from "@/components/Counter.tsx" import Counter

now = datetime.now().strftime("%H:%M:%S")
---
<div class="flex flex-col items-center justify-center min-h-screen bg-gray-900 text-white">
    <h1 class="text-5xl font-extrabold text-blue-400">Nexy Fullstack</h1>
    <p class="mt-4 text-xl">يقول الخادم أن الوقت الآن هو: <span class="font-mono text-yellow-400">{{ now }}</span></p>
    
    <!-- دمج مكون React عبر استيراد قياسي -->
    <div class="mt-8 p-6 bg-gray-800 rounded-xl shadow-2xl">
        <h2 class="text-lg mb-4">تفاعل العميل (React)</h2>
        <Counter />
    </div>
</div>
```

## 5. الأوركسترا في العمل

ابدأ الآن خادم التطوير الكامل:
```bash
nx dev
```
سيقوم Nexy الآن بـ:
1.  بدء واجهتك الخلفية بايثون (FastAPI).
2.  بدء منسق Vite (إذا تم تفعيله في nexyconfig.py).
3.  دمج الاثنين لتقديم صفحتك على العنوان `http://localhost:8000`.

---

## لماذا هذا التدفق مختلف؟

- **التدرج**: لست مضطرًا للقيام بـ fullstack. يمكن أن يكون Nexy واجهة خلفية محضة أو تطبيق SSR متجانس.
- **بدون كود لاصق (Zero Glue Code)**: لا توجد استدعاءات `fetch()` معقدة لبياناتك الأولية؛ فهي موجودة بالفعل، ومحددة الأنواع، في كتلة بايثون الخاصة بك عبر الـ `props`.
- **سرعة العرض**: يتم إنشاء HTML على الخادم، ويتولى Vite فقط التفاعل الغني من جانب العميل.

---

**نصيحة DX**: يستخدم Nexy نظام كاش ذكي لإعادة تجميع ما تغير فقط، مما يضمن أوقات تحديث أقل من 100 مللي ثانية.
