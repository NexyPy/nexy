# المكونات الأساسية: .nexy و .mdx

يقدم Nexy وحدتي تكوين رئيسيتين لبناء واجهات الاستخدام: ملفات **.nexy** وملفات **.mdx**.

## ملف .nexy

ملف `.nexy` هو **مكون خادم** يجمع بين منطق بايثون وقوالب HTML (Jinja2).

### الهيكل

ينقسم ملف `.nexy` إلى جزأين:

1.  **كتلة المنطق (بايثون)**: محصورة بين علامتي `---`.
2.  **كتلة العرض (قالب)**: HTML معزز بصيغة Jinja2.

```python
---
# منطق بايثون (جانب الخادم)
from pydantic import BaseModel

class UserProfile(BaseModel):
    id: int
    username: str

user : prop[UserProfile]
theme : prop[str] = "dark"

# استيراد مكونات العميل
from "@/components/Avatar.tsx" import Avatar  # مكون عميل
---
```

```html
<!-- قالب HTML/Jinja2 -->
<section class="profile-container {{ theme }}">
    <header>
        <Avatar src={user.avatar_url} />
        <h1>{{ user.username }}</h1>
    </header>
    
    <main>
        <p>معرف المستخدم: {{ user.id }}</p>
    </main>
</section>
```

### الخصائص الرئيسية

- **Pydantic Props**: قم بتعريف والتحقق من خصائصك باستخدام نماذج Pydantic.
- **حقن الاعتمادية**: استخدم نظام DI الخاص بـ Nexy لحقن الخدمات في كتلة بايثون.
- **عرض الخادم**: ينتج HTML نقيًا يتم تقديمه للعميل.

## دعم MDX

يدعم Nexy بشكل أصلي **MDX** ، والذي يسمح لك بكتابة Markdown مع مكونات مدمجة.

- **المحتوى أولاً**: مثالي للمدونات والتوثيق والصفحات الغنية بالمحتوى.
- **مدفوع بالمكونات**: استورد واستخدم مكونات React أو Vue أو Svelte أو Solid.js داخل Markdown الخاص بك.
- **صديق لمحركات البحث (SEO)**: يتم عرضه بتنسيق HTML على الخادم ، تمامًا مثل ملفات `.nexy`.

### مثال MDX

```mdx
# مقالي الأول في المدونة

أهلاً بك في مدونتي! إليك مكون React:

import Counter from '@/components/Counter.tsx'

<Counter initial={10} />

هذه فقرة بتنسيق **markdown**.
```

---

**نصيحة**: استخدم `.nexy` لمنطق التطبيق والتخطيطات المعقدة ، و `.mdx` للتوثيق ومحتوى المدونة.
