# التدويل (i18n)

يدمج Nexy نظام تدويل بسيط وعالي الأداء لجعل تطبيقاتك متاحة لجمهور عالمي.

## 1. هيكل الملفات

يتم تخزين ملفات الترجمة في دليل `locales/` (أو الدليل المحدد في التكوين الخاص بك) في شكل ملفات JSON.

```text
my-project/
├── locales/
│   ├── fr.json
│   └── en.json
└── src/
    └── routes/
        └── index.nexy
```

### مثال على ملف `fr.json`
```json
{
  "welcome": "مرحبًا بك في Nexy",
  "description": "إطار عمل بايثون متكامل"
}
```

## 2. الاستخدام في ملفات .nexy

يمكنك استخدام وظيفة `t()` (ترجمة) مباشرة في القوالب أو منطق بايثون الخاص بك.

```html
---
from nexy import t
---
<section>
    <h1>{{ t("welcome") }}</h1>
    <p>{{ t("description") }}</p>
</section>
```

## 3. اكتشاف اللغة

يكتشف Nexy تلقائيًا لغة المستخدم عبر:
1.  **معلمة URL**: `?lang=fr`
2.  **رأس HTTP**: `Accept-Language`
3.  **ملف تعريف الارتباط (Cookie)**: `nexy_locale`

## 4. الترجمات الديناميكية (المتغيرات)

يمكنك تمرير متغيرات إلى سلاسل النصوص الخاصة بك.

### ملف JSON
```json
{
  "greeting": "مرحبًا {name}!"
}
```

### الاستخدام
```python
{{ t("greeting", name="Alice") }}
```

## 5. التكوين

يمكنك تحديد اللغة الافتراضية في `nexyconfig.py`.

```python
from nexy.core.models import NexyConfigModel

class NexyConfig(NexyConfigModel):
    useLocaleDefault = "fr"
    useLocales = ["fr", "en"]
```

---

**نصيحة**: استخدم التدويل منذ بداية مشروعك لتسهيل توسيع تطبيقك في المستقبل.
