# Modular Router (App Router)

تم تصميم **Modular Router** (الموجود في `src/app/`) للبنيات المؤسسية. يعتمد على نظام **المتحكمات (Controllers)** و **الوحدات (Modules)** و **حقن الاعتمادية**.

على عكس أطر العمل الأخرى ، يستخدم Nexy نهجًا يعتمد على أساليب الفئات لتعريف المسارات ، بدلاً من المزخرفات لكل طريقة مثل `@Get`.

## 1. المتحكمات (Controllers)

**المتحكم** هو فئة مزخرفة بـ `@Controller()`. يتم تحديد المسارات من خلال **أسماء الأساليب** في الفئة المطابقة لأفعال HTTP (`get` ، `post` ، `put` ، `delete` ، إلخ.).

```python
# src/app/blog/blog_controller.py
from nexy.decorators import Controller

@Controller("/blog")
class BlogController:
    def __init__(self, service: BlogService):
        self.service = service

    # يحدد مسار GET /blog
    async def get(self):
        return await self.service.get_all()

    # يحدد مسار POST /blog
    async def post(self, data: dict):
        return await self.service.create(data)
```

### نقاط رئيسية للمتحكمات:
- **أسماء الأساليب**: استخدم `get` و `post` و `put` و `delete` و `patch` لتعريف نقاط نهاية HTTP.
- **WebSocket**: استخدم اسم الطريقة `socket` لتعريف نقطة دخول WebSocket.
- **معلمات المسار**: يتم ربط معلمات الطريقة تلقائيًا بمعلمات URL أو جسم الطلب (FastAPI).
- **الحقن التلقائي**: يتم حقن الخدمات الممررة إلى المنشئ `__init__` تلقائيًا إذا تم تمييزها كـ `@Injectable()`.

## 2. دعم WebSocket

يدمج Nexy الـ WebSockets أصليًا عبر Modular Router. ما عليك سوى تعريف طريقة تسمى `socket` في المتحكم الخاص بك.

```python
from nexy.decorators import Controller
from fastapi import WebSocket

@Controller("/ws")
class ChatController:
    async def socket(self, websocket: WebSocket):
        await websocket.accept()
        while True:
            data = await websocket.receive_text()
            await websocket.send_text(f"تم استلام الرسالة: {data}")
```

## 2. الوحدات (Modules)

**الوحدة** هي فئة مزخرفة بـ `@Module()` تقوم بتجميع المتحكمات والخدمات (providers) والوحدات الأخرى.

```python
# src/app/blog/blog_module.py
from nexy.decorators import Module
from .blog_controller import BlogController
from .blog_service import BlogService

@Module(prefix="/v1")
class BlogModule:
    controllers = [BlogController]
    providers = [BlogService]
```

### خصائص الوحدة:
- **controllers**: قائمة بفئات المتحكمات التي سيتم تسجيلها.
- **providers**: قائمة بفئات الخدمات التي سيتم جعلها قابلة للحقن.
- **imports**: قائمة بالوحدات الأخرى (الموجهات) التي سيتم تضمينها.

## 3. حقن الاعتمادية (DI)

لجعل الخدمة قابلة للحقن ، استخدم مزخرف `@Injectable()`.

```python
# src/app/blog/blog_service.py
from nexy.decorators import Injectable

@Injectable()
class BlogService:
    async def get_all(self):
        return [{"id": 1, "title": "Hello Nexy"}]
```

## 4. البيانات الوصفية للمسار (اختياري)

إذا كنت بحاجة إلى تخصيص بيانات OpenAPI الوصفية (العلامات ، الملخص ، إلخ) ، استخدم مزخرف `UseRoute`.

```python
from nexy.decorators import Controller, UseRoute

@Controller("/users")
class UserController:
    @UseRoute(summary="قائمة المستخدمين", tags=["Admin"])
    def get(self):
        return []
```
