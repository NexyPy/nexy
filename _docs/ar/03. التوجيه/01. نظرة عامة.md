# استراتيجيات التوجيه: Routes Router و Modular Router

يقدم Nexy استراتيجيتين رئيسيتين للتوجيه: **Routes Router** (القائم على الملفات) و **Modular Router** (القائم على الوحدات).

## Routes Router (القائم على الملفات)

يستخدم **Routes Router** الهيكل المادي لدليل `src/routes/` لتعريف عناوين URL الخاصة بتطبيقك تلقائيًا.

- **مدفوع بالطوبولوجيا**: تتوافق المجلدات والملفات مباشرة مع عناوين URL.
- **دعم لـ .nexy و .mdx و .py**: كل ملف في دليل `src/routes/` هو مسار.
- **التوجيه الديناميكي**: استخدم الأقواس المربعة `[]` للأجزاء الديناميكية (مثال: `src/routes/[id].nexy` -> `/{id}`).
- **صديق لمحركات البحث (SEO)**: يتم عرض كل مسار على جانب الخادم (SSR).

| الملف | عنوان URL |
| --- | --- |
| `src/routes/index.nexy` | `/` |
| `src/routes/docs/intro.mdx` | `/docs/intro` |
| `src/routes/api/health.py` | `/api/health` |

## Modular Router (القائم على الوحدات)

تم تصميم **Modular Router** (الموجود في `src/app/`) لواجهات البرمجة المعقدة وتطبيقات المؤسسات. يستخدم مزخرفات الفئات لإعلانات المسار الصريحة.

- **المعيارية**: قم بتنظيم تطبيقك في `Modules` و `Controllers` و `Providers`.
- **حقن الاعتمادية**: استخدم نظام DI الخاص بـ Nexy لإدارة خدماتك ومواردك.
- **الإعلانات الصريحة**: حدد مساراتك باستخدام مزخرفات مثل `@Controller` و `@Get` و `@Post` وما إلى ذلك.
- **قابلية الاختبار**: اختبر المتحكمات والخدمات الخاصة بك بسهولة وبشكل منفصل.

```python
@Controller("/api/v1/users")
class UserController:
    def __init__(self, user_service: UserService):
        self.service = user_service

    @Get("/{id}")
    async def get_profile(self, id: int):
        return await self.service.get_user(id)
```

## التوجيه الهجين

تكمن قوة Nexy في قدرته على الجمع بين هاتين الاستراتيجيتين:
1.  استخدم **Modular Router** للخدمات المصغرة وواجهات البرمجة ومنطق الأعمال.
2.  استخدم **Routes Router** لواجهات المستخدم والمحتوى والمدونات.

يسمح هذا الفصل بتصميم معماري نظيف مع الحفاظ على سرعة التطوير القائم على الملفات.

---

**نصيحة**: استخدم **Routes Router** للصفحات العامة و **Modular Router** للخدمات الداخلية وواجهات البرمجة.
