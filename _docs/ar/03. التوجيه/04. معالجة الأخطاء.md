# معالجة الأخطاء والتخطيطات

يوفر Nexy نظامًا قويًا لمعالجة الأخطاء وهيكلة صفحاتك باستخدام التخطيطات (layouts).

## 1. التخطيطات الهرمية (Layouts)

ملف `layout.nexy` في مجلد ما ينطبق تلقائيًا على جميع المسارات في هذا المجلد ومجلداته الفرعية.

### هيكل التخطيط
التخطيط هو ملف `.nexy` قياسي يجب أن يستخدم كتلة `{{ children }}` الخاصة لحقن محتوى الصفحات.

```html
---
# التخطيط الرئيسي
from nexy import Vite
---
<!DOCTYPE html>
<html lang="ar" dir="rtl">
<head>
    <meta charset="UTF-8">
    <title>تطبيقي في Nexy</title>
    {{ Vite() }}
</head>
<body class="bg-gray-100">
    <nav>...</nav>
    <main>
        {{ children }}
    </main>
    <footer>...</footer>
</body>
</html>
```

### تداخل التخطيطات
التخطيطات قابلة للتداخل. سيتم حقن تخطيط فرعي في `{{ children }}` الخاص بالتخطيط الوالد.

## 2. معالجة الأخطاء (404, 500)

يسمح Nexy بتخصيص صفحات الأخطاء بطريقة بسيطة.

### صفحة 404 (غير موجود)
قم بإنشاء ملف `not_found.nexy` في دليل `src/routes/` لتخصيص صفحة 404.

```html
---
# src/routes/not_found.nexy
---
<section class="flex flex-col items-center justify-center h-screen">
    <h1 class="text-4xl font-bold text-red-500">الصفحة غير موجودة</h1>
    <p class="mt-4">عذرًا ، المورد المطلوب غير موجود.</p>
    <a href="/" class="mt-6 text-blue-500 underline">العودة إلى الصفحة الرئيسية</a>
</section>
```

### صفحة الخطأ (500 ، إلخ.)
قم بإنشاء ملف `error.nexy` في دليل `src/routes/` لالتقاط استثناءات الخادم غير المتوقعة.

```html
---
# src/routes/error.nexy
status_code : prop[int]
detail : prop[str]
---
<section class="p-10 bg-red-50 border border-red-200 rounded-lg">
    <h2 class="text-2xl font-semibold text-red-700">خطأ في الخادم ({{ status_code }})</h2>
    <p class="mt-2 text-red-600">{{ detail }}</p>
</section>
```

## 3. استثناءات FastAPI

نظرًا لأن Nexy مبني على FastAPI ، يمكنك استخدام استثناءات FastAPI القياسية في ملفات `.py` أو المتحكمات الخاصة بك.

```python
from fastapi import HTTPException

async def GET():
    if not user_authenticated:
        raise HTTPException(status_code=401, detail="غير مصرح به")
    return {"data": "..."}
```

---

**نصيحة**: قم دائمًا بتعريف تخطيط جذر قوي لضمان تجربة مستخدم متسقة ، حتى في حالة حدوث خطأ.
