# Modular Router (App Router)

The **Modular Router** (located in `src/app/`) is designed for enterprise architectures. It relies on a system of **Controllers**, **Modules**, and **Dependency Injection**.

Unlike other frameworks, Nexy uses a class-method approach to define routes, rather than method decorators like `@Get`.

## 1. Controllers

A **Controller** is a class decorated with `@Controller()`. Routes are defined by the **method names** of the class, corresponding to HTTP verbs (`get`, `post`, `put`, `delete`, etc.).

```python
# src/app/blog/blog_controller.py
from nexy.decorators import Controller

@Controller("/blog")
class BlogController:
    def __init__(self, service: BlogService):
        self.service = service

    # Defines a GET /blog route
    async def get(self):
        return await self.service.get_all()

    # Defines a POST /blog route
    async def post(self, data: dict):
        return await self.service.create(data)
```

### Key Points of Controllers:
- **Method Names**: Use `get`, `post`, `put`, `delete`, `patch` to define HTTP endpoints.
- **WebSocket**: Use the method name `socket` to define a WebSocket entry point.
- **Route Parameters**: Method parameters are automatically bound to URL parameters or the request body (FastAPI).
- **Automatic Injection**: Services passed to the `__init__` constructor are automatically injected if marked as `@Injectable()`.

## 2. WebSocket Support

Nexy natively integrates WebSockets via the Modular Router. Simply define a method named `socket` in your controller.

```python
from nexy.decorators import Controller
from fastapi import WebSocket

@Controller("/ws")
class ChatController:
    async def socket(self, websocket: WebSocket):
        await websocket.accept()
        while True:
            data = await websocket.receive_text()
            await websocket.send_text(f"Message received: {data}")
```

## 2. Modules

A **Module** is a class decorated with `@Module()` that groups controllers, services (providers), and other modules.

```python
# src/app/blog/blog_module.py
from nexy.decorators import Module
from .blog_controller import BlogController
from .blog_service import BlogService

@Module(prefix="/v1")
class BlogModule:
    controllers = [BlogController]
    providers = [BlogService]
```

### Module Properties:
- **controllers**: List of controller classes to register.
- **providers**: List of service classes to make injectable.
- **imports**: List of other modules (routers) to include.

## 3. Dependency Injection (DI)

To make a service injectable, use the `@Injectable()` decorator.

```python
# src/app/blog/blog_service.py
from nexy.decorators import Injectable

@Injectable()
class BlogService:
    async def get_all(self):
        return [{"id": 1, "title": "Hello Nexy"}]
```

## 4. Route Metadata (Optional)

If you need to customize OpenAPI metadata (tags, summary, etc.), use the `UseRoute` decorator.

```python
from nexy.decorators import Controller, UseRoute

@Controller("/users")
class UserController:
    @UseRoute(summary="List users", tags=["Admin"])
    def get(self):
        return []
```
