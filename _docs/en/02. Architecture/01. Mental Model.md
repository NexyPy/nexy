# Mental Model: Nexy's Internal Workings

Nexy is a unified abstraction layer that synchronizes the server (Python/FastAPI) and the interface (HTML/JS). To use Nexy effectively, it is essential to understand its compilation process and rendering cycle.

## 1. Compilation Phase (Build Time)

When running `nx dev` or `nx build`, Nexy compiles your source files (`.nexy`, `.mdx`) into executable Python code:
- **AST Analysis (Abstract Syntax Tree)**: Nexy analyzes the logic block (Python) of your files to extract metadata (props, component imports).
- **Import Transformation**: Component imports (e.g., React, Vue, or other `.nexy` files) are transformed into calls to the internal `Import()` function. This function handles path resolution and data serialization.
- **Template Generation**: The HTML/Jinja2 template is compiled to transform component tags (PascalCase) into Python function calls.

## 2. Request Lifecycle (Runtime)

The technical steps when a page is accessed:
1.  **Server Execution**: Nexy executes the compiled Python code for the file corresponding to the route.
2.  **Props Validation**: Data passed to the component is validated via **Pydantic** based on the types defined in the logic block.
3.  **SSR Rendering (Jinja2)**: The engine generates the final HTML on the server.
4.  **Asset Injection**: The `Vite()` function (usually placed in the layout) injects the scripts required for frontend operation if the Vite orchestrator is enabled in the configuration.

## 3. Pure SSR Monolith (Vite-less)

Nexy is designed to function autonomously without dependency on Node.js or Vite if you are not using JavaScript frameworks:
- The interface is structured via `.nexy` server components (Python + HTML).
- Rendering is entirely managed by Python, offering maximum performance and optimal SEO.
- The Vite ecosystem is an interactivity extension that you enable only via `nexyconfig.py`.

## 4. Unified Routing Strategies

Nexy combines two complementary approaches for code organization:
- **File-Based Routing**: Uses the `src/routes/` directory structure to automatically define entry points (Pages, APIs, WebSockets).
- **Module-Based Routing**: Uses a class-based architecture (Modules, Controllers) for complex business services and dependency injection.

---

**Nexy Mental Model: A robust Python rendering engine that transforms polyglot components into pure HTML, with a seamless extension capability to JavaScript ecosystems via Vite.**
