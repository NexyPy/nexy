# Building Blocks: .nexy and .mdx

Nexy provides two primary units of composition for building interfaces: **.nexy** files and **.mdx** files.

## The .nexy File

A `.nexy` file is a **Server Component** that combines Python logic and HTML templates (Jinja2).

### Structure

A `.nexy` file is divided into two segments:

1.  **Logic Block (Python)**: Encapsulated between `---` markers.
2.  **Presentation Block (Template)**: HTML enriched with Jinja2 syntax.

```python
---
# Python Logic (Server-side)
from pydantic import BaseModel

class UserProfile(BaseModel):
    id: int
    username: str

user : prop[UserProfile]
theme : prop[str] = "dark"

# Import client components
from "@/components/Avatar.tsx" import Avatar  # Client component
---
```

```html
<!-- HTML/Jinja2 Template -->
<section class="profile-container {{ theme }}">
    <header>
        <Avatar src={user.avatar_url} />
        <h1>{{ user.username }}</h1>
    </header>
    
    <main>
        <p>User ID: {{ user.id }}</p>
    </main>
</section>
```

### Key Features

- **Pydantic Props**: Define and validate properties using Pydantic schemas.
- **Dependency Injection**: Use Nexy's DI system to inject services into the Python block.
- **Server Rendering**: Produces pure HTML that is served to the client.

## MDX Support

Nexy natively supports **MDX**, which allows you to write Markdown with embedded components.

- **Content-First**: Ideal for blogs, documentation, and content-rich pages.
- **Component-Driven**: Import and use React, Vue, Svelte, or Solid.js components within your Markdown.
- **SEO-Friendly**: Renders to HTML on the server, just like `.nexy` files.

### Example MDX

```mdx
# My First Blog Post

Welcome to my blog! Here's a React component:

import Counter from '@/components/Counter.tsx'

<Counter initial={10} />

This is a **markdown** paragraph.
```

---

**Tip**: Use `.nexy` for application logic and complex layouts, and `.mdx` for documentation and blog content.
