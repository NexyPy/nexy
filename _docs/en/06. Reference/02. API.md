# API Reference

This page provides a comprehensive list of all public APIs, decorators, and utilities available in Nexy.

## Decorators (`nexy.decorators`)

Nexy uses class-based decorators to structure your application and manage dependencies.

### `@Controller(prefix: str = "", tags: List[str] = None)`
Defines a routing controller. Methods within the class named after HTTP verbs (`get`, `post`, etc.) or `socket` for WebSockets automatically become endpoints.

### `@Module(prefix: str = "")`
Defines a module. Modules group controllers, providers, and other modules.
- `controllers`: List of controller classes.
- `providers`: List of injectable services.
- `imports`: List of other modules to include.

### `@Injectable()`
Marks a class as a service that can be injected into controllers or other services via the constructor.

### `UseRoute(...)`
Customizes OpenAPI metadata for a specific route (summary, description, tags, etc.).

### `UseResponse(...)`
Customizes the HTTP response metadata (status code, response model, etc.).

### `UseGuard(*guards)`
Attaches one or more guards to a controller or a specific route for authentication or authorization.

### `UseMiddleware(*middlewares)`
Attaches one or more middlewares to a controller or a specific route.

## Core Components & Utilities

### `Vite()`
Injects the necessary scripts for Vite (Hot Module Replacement in dev, asset links in prod). Usually handled automatically in layouts.

### `Template()`
A utility class for manual rendering of Jinja2 or Markdown templates.
- `render(path, context)`: Returns the rendered HTML string.

### `Import(path: str)`
Manually imports a client-side component (React, Vue, etc.). In `.nexy` files, this is handled automatically by the compiler.

### `Link()`
A utility function for generating consistent application links.

### `t(key: str, default: str = None, locale: str = None)`
Internationalization function. Fetches a translation string based on the current or specified locale.

## Multimedia & Forms

Nexy provides dedicated classes for handling common web elements. While currently serving as structured wrappers, they provide a foundation for future optimizations.

- **`Audio`**: Utilities for audio playback.
- **`Video`**: Utilities for video embedding.
- **`Image`**: Utilities for optimized image rendering.
- **`Form`**: Utilities for structured form handling.
- **`Script`**: Utilities for managing external or inline scripts.

---

## Hooks (`nexy.hooks`)

- **`useNexyRender(path)`**: Hook for custom rendering logic.
- **`useWebHook(path)`**: Hook for managing webhooks.
- **`UseStorage()`**: Hook for interacting with storage systems.

---

## Built-in Types

### `prop[T]`
Used within the Python block of a `.nexy` file to declare component properties with Pydantic validation.

```python
user : prop[UserSchema]
```

---

**Tip**: Always import these utilities directly from `nexy` or `nexy.decorators` to ensure compatibility with future updates.
