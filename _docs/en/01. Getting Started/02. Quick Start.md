# Quick Start: From API to Fullstack in 5 Minutes

This guide is designed for developers who want to quickly understand Nexy's flexibility, starting with a robust backend and ending with a hybrid interface.

## 1. Installation

```bash
pip install nexy
```

## 2. Initialize a "Pure Backend" Project

Let's start by creating a project without a frontend interface for now.

```bash
nx init my-project --no-frontend
cd my-project
```

## 3. Create a Modular API

Open `src/app/main.py` and define a simple controller:

```python
from nexy import Controller, Get

@Controller("/api")
class HelloController:
    @Get("/status")
    async def status(self):
        return {"status": "Nexy is running", "mode": "Pure Backend"}
```

Start the server:
```bash
nx start
```
Visit `http://localhost:8000/api/status`. You have an ultra-fast and modular FastAPI backend.

## 4. Migration to Fullstack (The "Wow" Effect)

Now, let's add a user interface without changing our backend. Simply create a `src/routes/index.nexy` file:

```html
---
# Integrated Python logic
from datetime import datetime
now = datetime.now().strftime("%H:%M:%S")
---
<div class="flex flex-col items-center justify-center min-h-screen bg-gray-900 text-white">
    <h1 class="text-5xl font-extrabold text-blue-400">Nexy Fullstack</h1>
    <p class="mt-4 text-xl">The server says it's: <span class="font-mono text-yellow-400">{{ now }}</span></p>
    
    <!-- Integrate a React component in 1 line -->
    <div class="mt-8 p-6 bg-gray-800 rounded-xl shadow-2xl">
        <h2 class="text-lg mb-4">Client Interactivity (React)</h2>
        <Counter client:load />
    </div>
</div>
```

## 5. Orchestration in Action

Now start the full development server:
```bash
nx dev
```
Nexy will now:
1.  Launch your Python backend.
2.  Launch the Vite orchestrator in the background.
3.  Merge both to serve your page at `http://localhost:8000`.

---

## Why is this flow different?

- **Progressive**: You aren't forced into fullstack.
- **Zero Glue Code**: No complicated `fetch()` calls for your initial data; it's already there, typed, in your Python block.
- **Rendering Speed**: HTML is generated on the server; Vite only handles interactivity.

---

**DX Tip**: Nexy uses an intelligent caching system to only re-compile what has changed, ensuring refresh times under 100ms.
