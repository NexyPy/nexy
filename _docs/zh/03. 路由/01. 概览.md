# 路由策略：Routes Router & Modular Router

Nexy 提供两种主要的路由策略：**Routes Router**（基于文件）和 **Modular Router**（基于模块）。

## Routes Router（基于文件）

**Routes Router** 使用 `src/routes/` 目录的物理结构来自动定义应用程序的 URL。

- **拓扑驱动**：文件夹和文件直接对应于 URL。
- **支持 .nexy, .mdx, .py**：`src/routes/` 目录中的每个文件都是一个路由。
- **动态路由**：使用方括号 `[]` 表示动态段（例如：`src/routes/[id].nexy` -> `/{id}`）。
- **SEO 友好**：每个路由都在服务器端渲染 (SSR)。

| 文件 | URL |
| --- | --- |
| `src/routes/index.nexy` | `/` |
| `src/routes/docs/intro.mdx` | `/docs/intro` |
| `src/routes/api/health.py` | `/api/health` |

## Modular Router（基于模块）

**Modular Router**（位于 `src/app/`）专为复杂的 API 和企业级应用程序设计。它使用类装饰器进行显式的路由声明。

- **模块化**：将您的应用程序组织为 `Modules`、`Controllers` 和 `Providers`。
- **依赖注入**：使用 Nexy 的 DI 系统来管理您的服务和资源。
- **显式声明**：使用 `@Controller`、`@Get`、`@Post` 等装饰器定义您的路由。
- **可测试性**：轻松地隔离测试您的控制器和服务。

```python
@Controller("/api/v1/users")
class UserController:
    def __init__(self, user_service: UserService):
        self.service = user_service

    @Get("/{id}")
    async def get_profile(self, id: int):
        return await self.service.get_user(id)
```

## 混合路由

Nexy 的强大之处在于能够结合这两种策略：
1.  使用 **Modular Router** 处理微服务、API 和业务逻辑。
2.  使用 **Routes Router** 处理用户界面、内容和博客。

这种分离允许清晰的架构设计，同时保持基于文件的快速开发。

---

**建议**：将 **Routes Router** 用于公共页面，将 **Modular Router** 用于内部服务和 API。
