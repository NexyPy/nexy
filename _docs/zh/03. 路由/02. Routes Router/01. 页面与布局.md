# Routes Router：页面、API 与 WebSocket

**Routes Router**（基于文件的路由）是 Nexy 的默认引擎。它使用 `src/routes/` 目录的结构来自动定义应用程序的入口点，无论是 HTML 页面、REST API 还是 WebSocket 连接。

## 1. 页面渲染 (UI)

`.nexy` 和 `.mdx` 文件会自动转换为 HTML 路由。

- `src/routes/index.nexy` -> `/`
- `src/routes/blog/[slug].mdx` -> `/blog/{slug}`

### 布局 (Layouts)
文件夹中的 `layout.nexy` 文件会自动应用于该文件夹及其子文件夹中的所有路由。

## 2. API 端点 (.py)

要创建纯 API 端点 (JSON)，请在 `src/routes/` 中使用 `.py` 文件。Nexy 通过**大写**的函数名来识别对应的 HTTP 方法。

```python
# src/routes/api/users.py
from pydantic import BaseModel

class User(BaseModel):
    id: int
    name: str

# GET /api/users 端点
async def GET():
    return [{"id": 1, "name": "Alice"}]

# POST /api/users 端点
async def POST(user: User):
    return {"status": "created", "user": user}
```

## 3. WebSocket 支持

Routes Router 还支持通过 `.py` 文件中名为 `SOCKET`（大写）的函数来实现 WebSocket。

```python
# src/routes/chat.py
from fastapi import WebSocket

async def SOCKET(websocket: WebSocket):
    await websocket.accept()
    await websocket.send_text("已连接到 Routes Router WebSocket！")
    await websocket.close()
```

## 4. 依赖管理

Nexy 为 Routes Router 提供了一个分层的依赖系统。在 `src/routes/` 的任何子目录中创建一个 `dependencies.py` 文件。

```python
# src/routes/admin/dependencies.py
from fastapi import Header, HTTPException

async def verify_token(x_token: str = Header(...)):
    if x_token != "fake-super-secret-token":
        raise HTTPException(status_code=400, detail="X-Token header invalid")

# /admin/ 及其子文件夹中的所有路由
# 都将使用这些依赖项。
dependencies = [verify_token]
```

### 依赖项特征：
- **继承**：定义在 `src/routes/` 根目录下的依赖项适用于整个应用程序。
- **累加**：子文件夹会将其自身的依赖项与其父文件夹的依赖项累加。
- **原生 FastAPI**：Nexy 内部使用 `fastapi.Depends`，因此您可以使用 FastAPI 的所有功能。

---

**建议**：将 `.py` 文件用于轻量级端点，将 **Modular Router** 用于需要类之间高级依赖注入的复杂服务。
