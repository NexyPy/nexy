# 错误处理与布局

Nexy 提供了一个强大的系统来处理错误并使用布局结构化您的页面。

## 1. 分层布局 (Layouts)

文件夹中的 `layout.nexy` 文件会自动应用于该文件夹及其子文件夹中的所有路由。

### 布局结构
布局是一个标准的 `.nexy` 文件，必须使用特殊的 `{{ children }}` 块来注入页面内容。

```html
---
# 主布局
from nexy import Vite
---
<!DOCTYPE html>
<html lang="zh">
<head>
    <meta charset="UTF-8">
    <title>我的 Nexy 应用程序</title>
    {{ Vite() }}
</head>
<body class="bg-gray-100">
    <nav>...</nav>
    <main>
        {{ children }}
    </main>
    <footer>...</footer>
</body>
</html>
```

### 布局嵌套
布局是可以嵌套的。子布局将被注入到父布局的 `{{ children }}` 中。

## 2. 错误处理 (404, 500)

Nexy 允许以简单的方式自定义错误页面。

### 404 页面（未找到）
在您的 `src/routes/` 目录中创建一个 `not_found.nexy` 文件来自定义 404 页面。

```html
---
# src/routes/not_found.nexy
---
<section class="flex flex-col items-center justify-center h-screen">
    <h1 class="text-4xl font-bold text-red-500">页面未找到</h1>
    <p class="mt-4">抱歉，请求的资源不存在。</p>
    <a href="/" class="mt-6 text-blue-500 underline">返回首页</a>
</section>
```

### 错误页面 (500 等)
在您的 `src/routes/` 目录中创建一个 `error.nexy` 文件，以捕获服务器意外的异常。

```html
---
# src/routes/error.nexy
status_code : prop[int]
detail : prop[str]
---
<section class="p-10 bg-red-50 border border-red-200 rounded-lg">
    <h2 class="text-2xl font-semibold text-red-700">服务器错误 ({{ status_code }})</h2>
    <p class="mt-2 text-red-600">{{ detail }}</p>
</section>
```

## 3. FastAPI 异常

由于 Nexy 构建在 FastAPI 之上，您可以在 `.py` 文件或控制器中使用标准的 FastAPI 异常。

```python
from fastapi import HTTPException

async def GET():
    if not user_authenticated:
        raise HTTPException(status_code=401, detail="未授权")
    return {"data": "..."}
```

---

**建议**：始终定义一个稳固的根布局，以确保即使在发生错误时也能提供一致的用户体验。
