# 配置 (`nexyconfig.py`)

Nexy 旨在以最小的配置运行（约定优于配置）。但是，对于复杂的项目，您可以通过项目根目录下的 `nexyconfig.py` 文件自定义框架的行为。

## 1. 基本结构

该文件必须定义一个继承自 `NexyConfigModel` 的 `NexyConfig` 类。

```python
from nexy.core.models import NexyConfigModel

class NexyConfig(NexyConfigModel):
    useTitle = "我的 Nexy 应用程序"
    useVite = True  # 激活 Vite 编排
```

## 2. 服务器选项

- **`usePort`** (`int`, 默认: `3000`)：开发和生产服务器监听的端口。
- **`useHost`** (`str`, 默认: `"0.0.0.0"`)：服务器监听的主机。
- **`useDocs`** (`bool`, 默认: `True`)：激活或停用 FastAPI 的 Swagger UI 界面。
- **`useDocsUrl`** (`str`, 默认: `"/docs"`)：Swagger 文档的访问 URL。
- **`useRedocsUrl`** (`str`, 默认: `"/redocs"`)：ReDoc 文档的访问 URL。

## 3. 前端编排 (Vite)

- **`useVite`** (`bool`, 默认: `False`)：激活 Vite 集成。如果您使用 React、Vue、Svelte 或 Solid.js 组件，则必选。
- **`useViteDevUrl`** (`str`, 可选)：Vite 开发服务器的自定义 URL（例如：`http://localhost:5173`）。
- **`useAliases`** (`dict`, 默认: `{}`)：为您的导入定义路径别名（例如：`{"@": "src"}`）。

## 4. 扩展与框架 (useFF)

Nexy 允许通过 `useFF` (Frontend Frameworks) 列表扩展其功能。

```python
from nexy.core.models import FFModel

class NexyConfig(NexyConfigModel):
    useFF = [
        FFModel(
            name="react",
            render="render_react", # 内部渲染函数
            extension=[".tsx", ".jsx"]
        )
    ]
```

## 5. 内容与 Markdown

- **`useMarkdownExtensions`** (`list[str]`, 默认: `[]`)：为 `.mdx` 文件渲染加载的 Python-Markdown 扩展列表（例如：`["extra", "codehilite"]`）。

## 6. 路由与排除

- **`excludeDirs`** (`list[str]`, 默认: `[]`)：自动路由发现（基于文件的路由）期间要忽略的目录列表。
- **`useRouter`** (`APIRouter`, 可选)：允许在启动时注入自定义的 `APIRouter` 实例。

---

**建议**：大多数选项都有合理的默认值。仅根据您的特定用例修改必要的内容。
