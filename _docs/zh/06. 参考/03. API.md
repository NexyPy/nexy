# API 参考

本页面列出了 Nexy 中可用的所有公共 API、装饰器和实用工具。

## 装饰器 (`nexy.decorators`)

Nexy 使用类装饰器来构建您的应用程序并管理依赖关系。

### `@Controller(prefix: str = "", tags: List[str] = None)`
定义一个路由控制器。类中以 HTTP 动词（`get`、`post` 等）命名的方法或用于 WebSocket 的 `socket` 方法将自动成为入口点 (endpoints)。

### `@Module(prefix: str = "")`
定义一个模块。模块组织控制器、提供者 (providers) 和其他模块。
- `controllers`：控制器类列表。
- `providers`：可注入服务列表。
- `imports`：要包含的其他模块列表。

### `@Injectable()`
将一个类标记为可以通过构造函数注入到控制器或其他服务中的服务。

### `UseRoute(...)`
为特定路由自定义 OpenAPI 元数据（摘要、描述、标签等）。

### `UseResponse(...)`
自定义 HTTP 响应元数据（状态码、响应模型等）。

### `UseGuard(*guards)`
将一个或多个“guards”附加到特定的控制器或路由，用于身份验证或授权。

### `UseMiddleware(*middlewares)`
将一个或多个中间件附加到特定的控制器或路由。

## 核心组件与实用工具

### `Vite()`
注入 Vite 所需的脚本（开发环境中的 HMR，生产环境中的资源链接）。通常在布局中自动处理。

### `Template()`
用于手动渲染 Jinja2 或 Markdown 模板的实用类。
- `render(path, context)`：返回渲染后的 HTML 字符串。

### `Import(path: str)`
手动导入客户端组件（React、Vue 等）。在 `.nexy` 文件中，这由编译器自动处理。

### `Link()`
用于生成一致的应用程序链接的实用函数。

### `t(key: str, default: str = None, locale: str = None)`
国际化函数。根据当前或指定的语言区域获取翻译字符串。

## 应用程序对象

### `app`
Nexy 使用的 **FastAPI** 原生实例。您可以使用它来注册全局中间件、自定义异常处理器或访问 FastAPI 的高级功能。

```python
from nexy import app

@app.on_event("startup")
async def startup_event():
    print("Nexy 应用程序已启动！")
```

## 多媒体与表单

Nexy 提供专门的类来管理常见的 Web 元素。虽然它们目前充当结构化的包装器，但它们为未来的优化奠定了基础。

- **`Audio`**：音频播放实用工具。
- **`Video`**：视频集成实用工具。
- **`Image`**：优化图像渲染的实用工具。
- **`Form`**：结构化表单管理的实用工具。
- **`Script`**：外部或内联脚本管理的实用工具。

---

## Hook (`nexy.hooks`)

- **`useNexyRender(path)`**：用于自定义渲染逻辑的 Hook。
- **`useWebHook(path)`**：用于 Webhook 管理的 Hook。
- **`UseStorage()`**：用于与存储系统交互的 Hook。

---

## 内置类型

### `prop[T]`
在 `.nexy` 文件的 Python 块中用于声明具有 Pydantic 验证的组件属性。

```python
user : prop[UserSchema]
```

---

**建议**：始终直接从 `nexy` 或 `nexy.decorators` 导入这些实用工具，以确保与未来更新的兼容性。
