# 快速开始：从 API 到全栈只需 5 分钟

本指南专为希望快速了解 Nexy 灵活性、从强大的后端开始并最终实现混合界面的开发者而设计。

## 1. 安装

```bash
pip install nexy
```

## 2. 初始化“纯后端”项目

让我们暂时创建一个没有前端界面的项目。

```bash
nx init my-project --no-frontend
cd my-project
```

## 3. 创建模块化 API

打开 `src/app/main.py` 并定义一个简单的控制器：

```python
from nexy.decorators import Controller

@Controller("/api")
class HelloController:
    # 方法名称定义 HTTP 动词（get、post 等）
    async def get(self):
        return {"status": "Nexy is running", "mode": "Pure Backend"}
```

启动服务器：
```bash
nx dev
```
访问 `http://localhost:8000/api`。您现在拥有一个超快且模块化的 FastAPI 后端。

## 4. 迁移到全栈（“惊艳”效果）

现在，让我们在不修改后端的情况下添加用户界面。只需创建一个 `src/routes/index.nexy` 文件：

```html
---
# 集成 Python 逻辑
from datetime import datetime
from "@/components/Counter.tsx" import Counter

now = datetime.now().strftime("%H:%M:%S")
---
<div class="flex flex-col items-center justify-center min-h-screen bg-gray-900 text-white">
    <h1 class="text-5xl font-extrabold text-blue-400">Nexy Fullstack</h1>
    <p class="mt-4 text-xl">服务器显示时间为：<span class="font-mono text-yellow-400">{{ now }}</span></p>
    
    <!-- 通过标准导入集成 React 组件 -->
    <div class="mt-8 p-6 bg-gray-800 rounded-xl shadow-2xl">
        <h2 class="text-lg mb-4">客户端交互 (React)</h2>
        <Counter />
    </div>
</div>
```

## 5. 编排实战

现在启动完整的开发服务器：
```bash
nx dev
```
Nexy 现在将：
1.  启动您的 Python 后端 (FastAPI)。
2.  启动 Vite 编排器（如果在 nexyconfig.py 中已启用）。
3.  合并两者以在 `http://localhost:8000` 提供页面服务。

---

## 为什么这种流程与众不同？

- **渐进式**：您不必非得做全栈。Nexy 可以是纯后端或 SSR 单体应用。
- **零胶水代码**：无需为初始数据编写复杂的 `fetch()` 调用；数据已经通过 `props` 存在于您的 Python 块中，且已类型化。
- **渲染速度**：HTML 在服务器上生成，Vite 仅处理丰富的客户端交互。

---

**DX 技巧**：Nexy 使用智能缓存系统仅重新编译已更改的内容，确保刷新时间低于 100ms。
