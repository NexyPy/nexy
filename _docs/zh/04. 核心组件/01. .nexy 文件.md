# 核心组件：.nexy 和 .mdx

Nexy 提供两个主要的组合单元来构建界面：**.nexy** 文件和 **.mdx** 文件。

## .nexy 文件

`.nexy` 文件是一个**服务端组件**，它结合了 Python 逻辑和 HTML 模板 (Jinja2)。

### 结构

`.nexy` 文件分为两个部分：

1.  **逻辑块 (Python)**：封装在 `---` 标记之间。
2.  **展示块 (Template)**：使用 Jinja2 语法增强的 HTML。

```python
---
# Python 逻辑（服务端）
from pydantic import BaseModel

class UserProfile(BaseModel):
    id: int
    username: str

user : prop[UserProfile]
theme : prop[str] = "dark"

# 导入客户端组件
from "@/components/Avatar.tsx" import Avatar  # 客户端组件
---
```

```html
<!-- HTML/Jinja2 模板 -->
<section class="profile-container {{ theme }}">
    <header>
        <Avatar src={user.avatar_url} />
        <h1>{{ user.username }}</h1>
    </header>
    
    <main>
        <p>用户 ID：{{ user.id }}</p>
    </main>
</section>
```

### 关键特性

- **Pydantic Props**：使用 Pydantic 模式定义并验证您的属性。
- **依赖注入**：使用 Nexy 的 DI 系统在 Python 块中注入服务。
- **服务端渲染**：生成提供给客户端的纯 HTML。

## MDX 支持

Nexy 原生支持 **MDX**，它允许您在 Markdown 中编写嵌入式组件。

- **内容优先**：非常适合博客、文档和内容丰富的页面。
- **组件驱动**：在您的 Markdown 中导入并使用 React、Vue、Svelte 或 Solid.js 组件。
- **SEO 友好**：在服务器上渲染为 HTML，就像 `.nexy` 文件一样。

### MDX 示例

```mdx
# 我的第一篇博客文章

欢迎来到我的博客！这是一个 React 组件：

import Counter from '@/components/Counter.tsx'

<Counter initial={10} />

这是一段 **markdown** 文本。
```

---

**建议**：将 `.nexy` 用于应用程序逻辑和复杂的布局，将 `.mdx` 用于文档和博客内容。
