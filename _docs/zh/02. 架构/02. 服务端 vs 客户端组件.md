# 服务端 vs 客户端组件

Nexy 使用混合组件模型。为了构建高性能应用程序，理解 **服务端组件** 和 **客户端组件** 之间的区别至关重要。

## 服务端组件 (.nexy, .mdx)

默认情况下，Nexy 中的每个文件都是 **服务端组件**。

- **执行**：它们仅在服务器上运行（Python）。
- **无 Bundle**：除非它们导入客户端组件，否则不会向浏览器发送 JavaScript。
- **SSR**：它们生成直接提供给浏览器的纯 HTML。
- **能力**：直接访问数据库、内部 API 和服务器密钥。

## 客户端组件 (.tsx, .vue, .svelte, .solid)

客户端组件用于为您的应用程序添加交互性。

- **执行**：它们由 **Vite** 打包并在客户端（浏览器）上激活（hydrated）。
- **交互性**：将它们用于状态管理、事件监听器或浏览器专用 API（例如 `localStorage`）。
- **隔离**：它们是您服务器渲染页面中的交互孤岛。

## Vite 编排

Nexy 中 Vite 的集成是通过项目配置管理的。它不是通过路由的静态分析自动激活的，而是通过您配置中的明确决定。

1.  **集中配置**：Vite 的激活是通过 `nexyconfig.py` 文件中的 `useVite: True` 完成的。
2.  **脚本注入**：如果该选项已启用，`Vite()` 函数（放在您的主 layout 中）会注入加载客户端组件所需的脚本。
3.  **自动激活**：当客户端组件（`.tsx`、`.vue` 等）导入到 `.nexy` 文件中时，Nexy 会通过 Vite 基础设施处理 props 序列化和客户端组件在浏览器中的挂载。不需要手动指令（例如 `client:load`）。

---

**建议**：仅当您需要 JavaScript 框架（React、Vue 等）时才启用 Vite。对于 100% Python SSR 应用程序，您可以禁用 Vite 以减少内存占用。
