# Configuration (`nexyconfig.py`)

Nexy est conçu pour fonctionner avec un minimum de configuration (Convention over Configuration). Cependant, pour les projets complexes, vous pouvez personnaliser le comportement du framework via le fichier `nexyconfig.py` à la racine de votre projet.

## 1. Structure de Base

Le fichier doit définir une classe `NexyConfig` héritant de `NexyConfigModel`.

```python
from nexy.core.models import NexyConfigModel

class NexyConfig(NexyConfigModel):
    useTitle = "Mon Application Nexy"
    useVite = True  # Active l'orchestration Vite
```

## 2. Options de Serveur

- **`usePort`** (`int`, défaut: `3000`) : Le port sur lequel le serveur de développement et de production écoutera.
- **`useHost`** (`str`, défaut: `"0.0.0.0"`) : L'hôte d'écoute du serveur.
- **`useDocs`** (`bool`, défaut: `True`) : Active ou désactive l'interface Swagger UI de FastAPI.
- **`useDocsUrl`** (`str`, défaut: `"/docs"`) : URL d'accès à la documentation Swagger.
- **`useRedocsUrl`** (`str`, défaut: `"/redocs"`) : URL d'accès à la documentation ReDoc.

## 3. Orchestration Frontend (Vite)

- **`useVite`** (`bool`, défaut: `False`) : Active l'intégration de Vite. Indispensable si vous utilisez des composants React, Vue, Svelte ou Solid.js.
- **`useViteDevUrl`** (`str`, optionnel) : URL personnalisée pour le serveur de développement Vite (ex: `http://localhost:5173`).
- **`useAliases`** (`dict`, défaut: `{}`) : Définit des alias de chemins pour vos imports (ex: `{"@": "src"}`).

## 4. Extensions & Frameworks (useFF)

Nexy permet d'étendre ses capacités via la liste `useFF` (Frontend Frameworks).

```python
from nexy.core.models import FFModel

class NexyConfig(NexyConfigModel):
    useFF = [
        FFModel(
            name="react",
            render="render_react", # Fonction de rendu interne
            extension=[".tsx", ".jsx"]
        )
    ]
```

## 5. Contenu & Markdown

- **`useMarkdownExtensions`** (`list[str]`, défaut: `[]`) : Liste des extensions Python-Markdown à charger pour le rendu des fichiers `.mdx` (ex: `["extra", "codehilite"]`).

## 6. Routage & Exclusion

- **`excludeDirs`** (`list[str]`, défaut: `[]`) : Liste des répertoires à ignorer lors de la découverte automatique des routes (File-Based Routing).
- **`useRouter`** (`APIRouter`, optionnel) : Permet d'injecter une instance `APIRouter` personnalisée au démarrage.

---

**Conseil** : La plupart des options ont des valeurs par défaut saines. Ne modifiez que ce qui est nécessaire pour votre cas d'usage spécifique.
