# Référence API

Cette page répertorie l'ensemble des APIs publiques, décorateurs et utilitaires disponibles dans Nexy.

## Décorateurs (`nexy.decorators`)

Nexy utilise des décorateurs de classe pour structurer votre application et gérer les dépendances.

### `@Controller(prefix: str = "", tags: List[str] = None)`
Définit un contrôleur de routage. Les méthodes de la classe nommées d'après les verbes HTTP (`get`, `post`, etc.) ou `socket` pour les WebSockets deviennent automatiquement des points d'entrée (endpoints).

### `@Module(prefix: str = "")`
Définit un module. Les modules regroupent des contrôleurs, des fournisseurs (providers) et d'autres modules.
- `controllers` : Liste des classes de contrôleurs.
- `providers` : Liste des services injectables.
- `imports` : Liste d'autres modules à inclure.

### `@Injectable()`
Marque une classe comme un service pouvant être injecté dans des contrôleurs ou d'autres services via le constructeur.

### `UseRoute(...)`
Personnalise les métadonnées OpenAPI pour une route spécifique (résumé, description, tags, etc.).

### `UseResponse(...)`
Personnalise les métadonnées de la réponse HTTP (code statut, modèle de réponse, etc.).

### `UseGuard(*guards)`
Attache un ou plusieurs "guards" à un contrôleur ou une route spécifique pour l'authentification ou l'autorisation.

### `UseMiddleware(*middlewares)`
Attache un ou plusieurs middlewares à un contrôleur ou une route spécifique.

## Composants de Base & Utilitaires

### `Vite()`
Injecte les scripts nécessaires pour Vite (HMR en développement, liens d'assets en production). Généralement géré automatiquement dans les layouts.

### `Template()`
Classe utilitaire pour le rendu manuel de templates Jinja2 ou Markdown.
- `render(path, context)` : Retourne la chaîne HTML rendue.

### `Import(path: str)`
Importe manuellement un composant côté client (React, Vue, etc.). Dans les fichiers `.nexy`, cela est géré automatiquement par le compilateur.

### `Link()`
Fonction utilitaire pour générer des liens d'application cohérents.

### `t(key: str, default: str = None, locale: str = None)`
Fonction d'internationalisation. Récupère une chaîne de traduction basée sur la locale actuelle ou spécifiée.

## L'Objet Application

### `app`
L'instance native de **FastAPI** utilisée par Nexy. Vous pouvez l'utiliser pour enregistrer des middlewares globaux, des gestionnaires d'exceptions personnalisés ou accéder aux fonctionnalités avancées de FastAPI.

```python
from nexy import app

@app.on_event("startup")
async def startup_event():
    print("Application Nexy démarrée !")
```

## Multimédia & Formulaires

Nexy fournit des classes dédiées pour gérer les éléments web courants. Bien qu'elles servent actuellement de wrappers structurés, elles constituent une base pour de futures optimisations.

- **`Audio`** : Utilitaires pour la lecture audio.
- **`Video`** : Utilitaires pour l'intégration de vidéos.
- **`Image`** : Utilitaires pour le rendu d'images optimisées.
- **`Form`** : Utilitaires pour la gestion structurée des formulaires.
- **`Script`** : Utilitaires pour la gestion des scripts externes ou en ligne.

---

## Hooks (`nexy.hooks`)

- **`useNexyRender(path)`** : Hook pour la logique de rendu personnalisée.
- **`useWebHook(path)`** : Hook pour la gestion des webhooks.
- **`UseStorage()`** : Hook pour l'interaction avec les systèmes de stockage.

---

## Types Intégrés

### `prop[T]`
Utilisé dans le bloc Python d'un fichier `.nexy` pour déclarer les propriétés du composant avec validation Pydantic.

```python
user : prop[UserSchema]
```

---

**Conseil** : Importez toujours ces utilitaires directement depuis `nexy` ou `nexy.decorators` pour garantir la compatibilité avec les futures mises à jour.
