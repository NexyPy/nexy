# Gestion des Erreurs et Layouts

Nexy propose un système robuste pour gérer les erreurs et structurer vos pages avec des layouts.

## 1. Layouts Hiérarchiques

Un fichier `layout.nexy` dans un dossier s'applique automatiquement à toutes les routes de ce dossier et de ses sous-dossiers.

### Structure d'un Layout
Un layout est un fichier `.nexy` standard qui doit utiliser le bloc spécial `{{ children }}` pour injecter le contenu des pages.

```html
---
# Layout principal
from nexy import Vite
---
<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <title>Mon Application Nexy</title>
    {{ Vite() }}
</head>
<body class="bg-gray-100">
    <nav>...</nav>
    <main>
        {{ children }}
    </main>
    <footer>...</footer>
</body>
</html>
```

### Emboîtement des Layouts
Les layouts sont emboîtés. Un layout enfant sera injecté dans le `{{ children }}` du layout parent.

## 2. Gestion des Erreurs (404, 500)

Nexy permet de personnaliser les pages d'erreur de manière simple.

### Page 404 (Non Trouvé)
Créez un fichier `not_found.nexy` dans votre répertoire `src/routes/` pour personnaliser la page 404.

```html
---
# src/routes/not_found.nexy
---
<section class="flex flex-col items-center justify-center h-screen">
    <h1 class="text-4xl font-bold text-red-500">Page Introuvable</h1>
    <p class="mt-4">Désolé, la ressource demandée n'existe pas.</p>
    <a href="/" class="mt-6 text-blue-500 underline">Retour à l'accueil</a>
</section>
```

### Page d'Erreur (500, etc.)
Créez un fichier `error.nexy` dans votre répertoire `src/routes/` pour capturer les exceptions inattendues du serveur.

```html
---
# src/routes/error.nexy
status_code : prop[int]
detail : prop[str]
---
<section class="p-10 bg-red-50 border border-red-200 rounded-lg">
    <h2 class="text-2xl font-semibold text-red-700">Erreur du Serveur ({{ status_code }})</h2>
    <p class="mt-2 text-red-600">{{ detail }}</p>
</section>
```

## 3. Exceptions FastAPI

Puisque Nexy est construit sur FastAPI, vous pouvez utiliser les exceptions standard de FastAPI dans vos fichiers `.py` ou vos contrôleurs.

```python
from fastapi import HTTPException

async def GET():
    if not user_authenticated:
        raise HTTPException(status_code=401, detail="Non autorisé")
    return {"data": "..."}
```

---

**Conseil** : Définissez toujours un layout racine solide pour assurer une expérience utilisateur cohérente, même en cas d'erreur.
