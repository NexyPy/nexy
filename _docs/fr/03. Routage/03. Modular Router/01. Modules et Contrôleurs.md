# Modular Router (App Router)

Le **Modular Router** (situé dans `src/app/`) est conçu pour les architectures d'entreprise. Il repose sur un système de **Contrôleurs**, de **Modules** et d'**Injection de Dépendances**.

Contrairement à d'autres frameworks, Nexy utilise une approche basée sur les méthodes de classe pour définir les routes, plutôt que des décorateurs par méthode comme `@Get`.

## 1. Contrôleurs (Controllers)

Un **Contrôleur** est une classe décorée avec `@Controller()`. Les routes sont définies par le **nom des méthodes** de la classe correspondant aux verbes HTTP (`get`, `post`, `put`, `delete`, etc.).

```python
# src/app/blog/blog_controller.py
from nexy.decorators import Controller

@Controller("/blog")
class BlogController:
    def __init__(self, service: BlogService):
        self.service = service

    # Définit une route GET /blog
    async def get(self):
        return await self.service.get_all()

    # Définit une route POST /blog
    async def post(self, data: dict):
        return await self.service.create(data)
```

### Points clés des Contrôleurs :
- **Noms de méthodes** : Utilisez `get`, `post`, `put`, `delete`, `patch` pour définir les endpoints HTTP.
- **WebSocket** : Utilisez le nom de méthode `socket` pour définir un point d'entrée WebSocket.
- **Paramètres de route** : Les paramètres de la méthode sont automatiquement liés aux paramètres de l'URL ou au corps de la requête (FastAPI).
- **Injection automatique** : Les services passés au constructeur `__init__` sont injectés automatiquement s'ils sont marqués comme `@Injectable()`.

## 2. Support WebSocket

Nexy intègre nativement les WebSockets via le Modular Router. Il suffit de définir une méthode nommée `socket` dans votre contrôleur.

```python
from nexy.decorators import Controller
from fastapi import WebSocket

@Controller("/ws")
class ChatController:
    async def socket(self, websocket: WebSocket):
        await websocket.accept()
        while True:
            data = await websocket.receive_text()
            await websocket.send_text(f"Message reçu: {data}")
```

## 2. Modules

Un **Module** est une classe décorée avec `@Module()` qui regroupe les contrôleurs, les services (providers) et d'autres modules.

```python
# src/app/blog/blog_module.py
from nexy.decorators import Module
from .blog_controller import BlogController
from .blog_service import BlogService

@Module(prefix="/v1")
class BlogModule:
    controllers = [BlogController]
    providers = [BlogService]
```

### Propriétés du Module :
- **controllers** : Liste des classes de contrôleurs à enregistrer.
- **providers** : Liste des classes de services à rendre injectables.
- **imports** : Liste d'autres modules (routeurs) à inclure.

## 3. Injection de Dépendances (DI)

Pour rendre un service injectable, utilisez le décorateur `@Injectable()`.

```python
# src/app/blog/blog_service.py
from nexy.decorators import Injectable

@Injectable()
class BlogService:
    async def get_all(self):
        return [{"id": 1, "title": "Hello Nexy"}]
```

## 4. Métadonnées de Route (Optionnel)

Si vous avez besoin de personnaliser les métadonnées OpenAPI (tags, summary, etc.), utilisez le décorateur `UseRoute`.

```python
from nexy.decorators import Controller, UseRoute

@Controller("/users")
class UserController:
    @UseRoute(summary="Liste des utilisateurs", tags=["Admin"])
    def get(self):
        return []
```

---

**Note technique** : Le Modular Router de Nexy transforme votre structure de classes en un `APIRouter` FastAPI de manière transparente, garantissant des performances optimales.
