# Routes Router : Pages, APIs & WebSockets

Le **Routes Router** (File-Based Routing) est le moteur par défaut de Nexy. Il utilise la structure de votre répertoire `src/routes/` pour définir automatiquement les points d'entrée de votre application, qu'il s'agisse de pages HTML, d'APIs REST ou de connexions WebSocket.

## 1. Rendu de Pages (UI)

Les fichiers `.nexy` et `.mdx` sont automatiquement transformés en routes HTML.

- `src/routes/index.nexy` -> `/`
- `src/routes/blog/[slug].mdx` -> `/blog/{slug}`

### Layouts
Un fichier `layout.nexy` dans un dossier s'applique automatiquement à toutes les routes de ce dossier et de ses sous-dossiers.

## 2. Endpoints API (.py)

Pour créer des endpoints API pur (JSON), utilisez des fichiers `.py` dans `src/routes/`. Nexy identifie les fonctions dont le nom correspond aux verbes HTTP en **majuscules**.

```python
# src/routes/api/users.py
from pydantic import BaseModel

class User(BaseModel):
    id: int
    name: str

# Endpoint GET /api/users
async def GET():
    return [{"id": 1, "name": "Alice"}]

# Endpoint POST /api/users
async def POST(user: User):
    return {"status": "created", "user": user}
```

## 3. Support WebSocket

Le Routes Router supporte également les WebSockets via une fonction nommée `SOCKET` (en majuscules) dans un fichier `.py`.

```python
# src/routes/chat.py
from fastapi import WebSocket

async def SOCKET(websocket: WebSocket):
    await websocket.accept()
    await websocket.send_text("Connecté au Routes Router WebSocket !")
    await websocket.close()
```

## 4. Gestion des Dépendances

Nexy propose un système de dépendances hiérarchique pour le Routes Router. Créez un fichier `dependencies.py` dans n'importe quel sous-répertoire de `src/routes/`.

```python
# src/routes/admin/dependencies.py
from fastapi import Header, HTTPException

async def verify_token(x_token: str = Header(...)):
    if x_token != "fake-super-secret-token":
        raise HTTPException(status_code=400, detail="X-Token header invalid")

# Toutes les routes dans /admin/ et ses sous-dossiers
# utiliseront ces dépendances.
dependencies = [verify_token]
```

### Caractéristiques des dépendances :
- **Héritage** : Les dépendances définies à la racine de `src/routes/` s'appliquent à toute l'application.
- **Cumul** : Un dossier enfant cumule ses propres dépendances avec celles de ses parents.
- **FastAPI Native** : Nexy utilise `fastapi.Depends` en interne, vous pouvez donc utiliser toutes les fonctionnalités de FastAPI.

---

**Conseil** : Utilisez les fichiers `.py` pour vos endpoints légers et le **Modular Router** pour vos services complexes nécessitant une injection de dépendances avancée entre classes.
