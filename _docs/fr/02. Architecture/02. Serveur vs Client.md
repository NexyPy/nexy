# Composants Serveur vs Client

Nexy utilise un modèle de composants hybrides. Pour construire des applications performantes, il est crucial de comprendre la différence entre les **Composants Serveur** et les **Composants Client**.

## Composants Serveur (.nexy, .mdx)

Par défaut, chaque fichier dans Nexy est un **Composant Serveur**.

- **Exécution** : Ils s'exécutent exclusivement sur le serveur (Python).
- **Pas de Bundle** : Ils n'envoient pas de JavaScript au navigateur sauf s'ils importent des composants client.
- **SSR** : Ils produisent du HTML pur qui est servi directement au navigateur.
- **Capacités** : Accès direct aux bases de données, aux APIs internes et aux secrets du serveur.

## Composants Client (.tsx, .vue, .svelte, .solid)

Les composants client sont utilisés pour ajouter de l'interactivité à votre application.

- **Exécution** : Ils sont regroupés par **Vite** et hydratés sur le client (navigateur).
- **Interactivité** : Utilisez-les pour la gestion de l'état, les écouteurs d'événements ou les APIs réservées au navigateur (ex: `localStorage`).
- **Isolation** : Ce sont des îlots d'interactivité isolés au sein de vos pages rendues par le serveur.

## Orchestration de Vite

L'intégration de Vite dans Nexy est gérée via la configuration du projet. Elle n'est pas activée automatiquement par analyse statique des routes, mais par une décision explicite dans votre configuration.

1.  **Configuration Centralisée** : L'activation de Vite se fait via `useVite: True` dans votre fichier `nexyconfig.py`.
2.  **Injection des Scripts** : La fonction `Vite()` (placée dans votre layout principal) injecte les scripts nécessaires au chargement des composants clients si l'option est activée.
3.  **Hydratation Automatique** : Lorsqu'un composant client (`.tsx`, `.vue`, etc.) est importé dans un fichier `.nexy`, Nexy gère la sérialisation des props et le montage du composant dans le navigateur via l'infrastructure Vite. Aucune directive manuelle (ex: `client:load`) n'est requise.

---

**Conseil** : Activez Vite uniquement si vous avez besoin de frameworks JavaScript (React, Vue, etc.). Pour une application 100% Python SSR, vous pouvez désactiver Vite pour une empreinte mémoire réduite.
