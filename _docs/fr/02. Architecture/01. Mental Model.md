# Mental Model : Fonctionnement Interne de Nexy

Nexy est une couche d'abstraction unifiée qui synchronise le serveur (Python/FastAPI) et l'interface (HTML/JS). Pour utiliser Nexy efficacement, il est essentiel de comprendre son processus de compilation et son cycle de rendu.

## 1. Phase de Compilation (Build Time)

Lors de l'exécution de `nx dev` ou `nx build`, Nexy compile vos fichiers source (`.nexy`, `.mdx`) pour les transformer en code Python exécutable :
- **Analyse AST (Abstract Syntax Tree)** : Nexy analyse le bloc logique (Python) de vos fichiers pour extraire les métadonnées (props, imports de composants).
- **Transformation des Imports** : Les imports de composants (ex: React, Vue ou autres fichiers `.nexy`) sont transformés en appels à la fonction interne `Import()`. Cette fonction gère la résolution des chemins et la sérialisation des données.
- **Génération de Template** : Le template HTML/Jinja2 est compilé pour transformer les balises de composants (PascalCase) en appels de fonctions Python.

## 2. Le Cycle de Vie d'une Requête (Runtime)

Voici les étapes techniques lors de la consultation d'une page :
1.  **Exécution Serveur** : Nexy exécute le code Python compilé du fichier correspondant à la route.
2.  **Validation des Props** : Les données passées au composant sont validées via **Pydantic** selon les types définis dans le bloc logique.
3.  **Rendu SSR (Jinja2)** : Le moteur génère le HTML final sur le serveur.
4.  **Injection d'Assets** : La fonction `Vite()` (généralement placée dans le layout) injecte les scripts nécessaires au fonctionnement du frontend si l'orchestrateur Vite est activé dans la configuration.

## 3. Rendu Monolithe SSR (Vite-less)

Nexy est conçu pour fonctionner de manière autonome sans dépendance à Node.js ou Vite si vous n'utilisez pas de frameworks JavaScript :
- L'interface est structurée via des composants serveur `.nexy` (Python + HTML).
- Le rendu est intégralement géré par Python, offrant des performances maximales et un SEO optimal.
- L'écosystème Vite est une extension d'interactivité que vous activez uniquement via `nexyconfig.py`.

## 4. Stratégies de Routage Unifiées

Nexy combine deux approches complémentaires pour l'organisation du code :
- **File-Based Routing** : Utilise la structure du répertoire `src/routes/` pour définir automatiquement les points d'entrée (Pages, APIs, WebSockets).
- **Module-Based Routing** : Utilise une architecture par classes (Modules, Controllers) pour les services métier complexes et l'injection de dépendances.

---

**Le Mental Model de Nexy : Un moteur de rendu Python robuste qui transforme des composants polyglottes en HTML pur, avec une capacité d'extension transparente vers les écosystèmes JavaScript via Vite.**
