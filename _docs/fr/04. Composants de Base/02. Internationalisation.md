# Internationalisation (i18n)

Nexy intègre un système d'internationalisation simple et performant pour rendre vos applications accessibles à un public mondial.

## 1. Structure des Fichiers

Les fichiers de traduction sont stockés dans un répertoire `locales/` (ou celui défini dans votre configuration) sous forme de fichiers JSON.

```text
my-project/
├── locales/
│   ├── fr.json
│   └── en.json
└── src/
    └── routes/
        └── index.nexy
```

### Exemple de fichier `fr.json`
```json
{
  "welcome": "Bienvenue sur Nexy",
  "description": "Le framework Python fullstack"
}
```

## 2. Utilisation dans les fichiers .nexy

Vous pouvez utiliser la fonction `t()` (translate) directement dans vos templates ou votre logique Python.

```html
---
from nexy import t
---
<section>
    <h1>{{ t("welcome") }}</h1>
    <p>{{ t("description") }}</p>
</section>
```

## 3. Détection de la Langue

Nexy détecte automatiquement la langue de l'utilisateur via :
1.  **Le paramètre d'URL** : `?lang=fr`
2.  **Le Header HTTP** : `Accept-Language`
3.  **Le Cookie** : `nexy_locale`

## 4. Traductions Dynamiques (Variables)

Vous pouvez passer des variables à vos chaînes de caractères.

### Fichier JSON
```json
{
  "greeting": "Bonjour {name} !"
}
```

### Usage
```python
{{ t("greeting", name="Alice") }}
```

## 5. Configuration

Vous pouvez définir la langue par défaut dans votre `nexyconfig.py`.

```python
from nexy.core.models import NexyConfigModel

class NexyConfig(NexyConfigModel):
    useLocaleDefault = "fr"
    useLocales = ["fr", "en"]
```

---

**Conseil** : Utilisez l'internationalisation dès le début de votre projet pour faciliter l'expansion future de votre application.
