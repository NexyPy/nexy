# Building Blocks : .nexy et .mdx

Nexy propose deux unités principales de composition pour construire des interfaces : les fichiers **.nexy** et les fichiers **.mdx**.

## Le Fichier .nexy

Un fichier `.nexy` est un **Composant Serveur** qui combine logique Python et templates HTML (Jinja2).

### Structure

Un fichier `.nexy` est divisé en deux segments :

1.  **Bloc Logique (Python)** : Encapsulé entre les marqueurs `---`.
2.  **Bloc de Présentation (Template)** : HTML enrichi avec la syntaxe Jinja2.

```python
---
# Logique Python (Côté Serveur)
from pydantic import BaseModel

class UserProfile(BaseModel):
    id: int
    username: str

user : prop[UserProfile]
theme : prop[str] = "dark"

# Import de composants client
from "@/components/Avatar.tsx" import Avatar  # Composant Client
---
```

```html
<!-- HTML/Jinja2 Template -->
<section class="profile-container {{ theme }}">
    <header>
        <Avatar src={user.avatar_url} />
        <h1>{{ user.username }}</h1>
    </header>
    
    <main>
        <p>ID Utilisateur : {{ user.id }}</p>
    </main>
</section>
```

### Caractéristiques Clés

- **Pydantic Props** : Définissez et validez vos propriétés avec des schémas Pydantic.
- **Injection de Dépendances** : Utilisez le système DI de Nexy pour injecter des services dans le bloc Python.
- **Rendu Serveur** : Produit du HTML pur qui est servi au client.

## Support MDX

Nexy supporte nativement **MDX**, qui permet d'écrire du Markdown avec des composants intégrés.

- **Contenu-First** : Idéal pour les blogs, la documentation et les pages riches en contenu.
- **Composant-Driven** : Importez et utilisez des composants React, Vue, Svelte ou Solid.js dans votre Markdown.
- **SEO-Friendly** : Rendu en HTML sur le serveur, tout comme les fichiers `.nexy`.

### Exemple MDX

```mdx
# Mon Premier Article de Blog

Bienvenue sur mon blog ! Voici un composant React :

import Counter from '@/components/Counter.tsx'

<Counter initial={10} />

C'est un paragraphe en **markdown**.
```

---

**Conseil** : Utilisez `.nexy` pour la logique d'application et les layouts complexes, et `.mdx` pour la documentation et le contenu de blog.
