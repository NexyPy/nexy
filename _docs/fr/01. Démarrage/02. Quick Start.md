# Quick Start : De l'API au Fullstack en 5 Minutes

Ce guide est conçu pour les développeurs qui veulent comprendre rapidement la flexibilité de Nexy, en commençant par un backend robuste pour finir sur une interface hybride.

## 1. Installation

```bash
pip install nexy
```

## 2. Initialisation d'un projet "Pure Backend"

Commençons par créer un projet sans interface frontend pour le moment.

```bash
nx init my-project --no-frontend
cd my-project
```

## 3. Création d'une API Modulaire

Ouvrez `src/app/main.py` et définissez un contrôleur simple :

```python
from nexy.decorators import Controller

@Controller("/api")
class HelloController:
    # Le nom de la méthode définit le verbe HTTP (get, post, etc.)
    async def get(self):
        return {"status": "Nexy is running", "mode": "Pure Backend"}
```

Lancez le serveur :
```bash
nx dev
```
Visitez `http://localhost:8000/api`. Vous avez un backend FastAPI ultra-rapide et modulaire.

## 4. Migration vers le Fullstack (L'effet "Wow")

Maintenant, ajoutons une interface utilisateur sans modifier notre backend. Créez simplement un fichier `src/routes/index.nexy` :

```html
---
# Logique Python intégrée
from datetime import datetime
from "@/components/Counter.tsx" import Counter

now = datetime.now().strftime("%H:%M:%S")
---
<div class="flex flex-col items-center justify-center min-h-screen bg-gray-900 text-white">
    <h1 class="text-5xl font-extrabold text-blue-400">Nexy Fullstack</h1>
    <p class="mt-4 text-xl">Le serveur dit qu'il est : <span class="font-mono text-yellow-400">{{ now }}</span></p>
    
    <!-- Intégration d'un composant React via un import standard -->
    <div class="mt-8 p-6 bg-gray-800 rounded-xl shadow-2xl">
        <h2 class="text-lg mb-4">Interactivité Client (React)</h2>
        <Counter />
    </div>
</div>
```

## 5. L'Orchestration en Action

Lancez maintenant le serveur de développement complet :
```bash
nx dev
```
Nexy va maintenant :
1.  Lancer votre backend Python (FastAPI).
2.  Lancer l'orchestrateur Vite (si activé dans nexyconfig.py).
3.  Fusionner les deux pour servir votre page à l'adresse `http://localhost:8000`.

---

## Pourquoi ce flux est-il différent ?

- **Progressivité** : Vous n'êtes pas forcé de faire du fullstack. Nexy peut être un backend pur ou un monolithe SSR.
- **Zéro Glue Code** : Pas d'appels `fetch()` compliqués pour vos données initiales ; elles sont déjà là, typées, dans votre bloc Python via les `props`.
- **Vitesse de Rendu** : Le HTML est généré sur le serveur, Vite ne s'occupe que de l'interactivité riche côté client.

---

**Astuce DX** : Nexy utilise un système de cache intelligent pour ne re-compiler que ce qui a changé, garantissant des temps de rafraîchissement inférieurs à 100ms.
