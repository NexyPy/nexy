# त्रुटि प्रबंधन और लेआउट

Nexy त्रुटियों को संभालने और लेआउट के साथ आपके पृष्ठों को संरचित करने के लिए एक मजबूत प्रणाली प्रदान करता है।

## 1. पदानुक्रमित लेआउट (Layouts)

किसी फोल्डर में `layout.nexy` फ़ाइल उस फोल्डर और उसके सब-फोल्डers के सभी रूट पर स्वचालित रूप से लागू होती है।

### लेआउट की संरचना
एक लेआउट एक मानक `.nexy` फ़ाइल है जिसे पृष्ठों की सामग्री को इंजेक्ट करने के लिए विशेष `{{ children }}` ब्लॉक का उपयोग करना चाहिए।

```html
---
# मुख्य लेआउट
from nexy import Vite
---
<!DOCTYPE html>
<html lang="hi">
<head>
    <meta charset="UTF-8">
    <title>मेरा Nexy एप्लिकेशन</title>
    {{ Vite() }}
</head>
<body class="bg-gray-100">
    <nav>...</nav>
    <main>
        {{ children }}
    </main>
    <footer>...</footer>
</body>
</html>
```

### लेआउट का नेस्टिंग
लेआउट नेस्टेबल हैं। एक चाइल्ड लेआउट पैरेंट लेआउट के `{{ children }}` में इंजेक्ट किया जाएगा।

## 2. त्रुटि प्रबंधन (404, 500)

Nexy त्रुटि पृष्ठों को सरल तरीके से अनुकूलित करने की अनुमति देता है।

### 404 पृष्ठ (नहीं मिला)
404 पृष्ठ को अनुकूलित करने के लिए अपनी `src/routes/` निर्देशिका में एक `not_found.nexy` फ़ाइल बनाएँ।

```html
---
# src/routes/not_found.nexy
---
<section class="flex flex-col items-center justify-center h-screen">
    <h1 class="text-4xl font-bold text-red-500">पृष्ठ नहीं मिला</h1>
    <p class="mt-4">क्षमा करें, अनुरोधित संसाधन मौजूद नहीं है।</p>
    <a href="/" class="mt-6 text-blue-500 underline">होम पर वापस जाएँ</a>
</section>
```

### त्रुटि पृष्ठ (500, आदि)
अप्रत्याशित सर्वर अपवादों को पकड़ने के लिए अपनी `src/routes/` निर्देशिका में एक `error.nexy` फ़ाइल बनाएँ।

```html
---
# src/routes/error.nexy
status_code : prop[int]
detail : prop[str]
---
<section class="p-10 bg-red-50 border border-red-200 rounded-lg">
    <h2 class="text-2xl font-semibold text-red-700">सर्वर त्रुटि ({{ status_code }})</h2>
    <p class="mt-2 text-red-600">{{ detail }}</p>
</section>
```

## 3. FastAPI अपवाद

चूँकि Nexy FastAPI पर बनाया गया है, आप अपनी `.py` फ़ाइलों या कंट्रोलर्स में मानक FastAPI अपवादों का उपयोग कर सकते हैं।

```python
from fastapi import HTTPException

async def GET():
    if not user_authenticated:
        raise HTTPException(status_code=401, detail="अनधिकृत")
    return {"data": "..."}
```

---

**सलाह**: त्रुटि होने पर भी एक सुसंगत उपयोगकर्ता अनुभव सुनिश्चित करने के लिए हमेशा एक ठोस रूट लेआउट परिभाषित करें।
