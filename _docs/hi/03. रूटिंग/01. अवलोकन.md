# रूटिंग रणनीतियाँ: Routes Router और Modular Router

Nexy दो मुख्य रूटिंग रणनीतियाँ प्रदान करता है: **Routes Router** (फ़ाइल-आधारित) और **Modular Router** (मॉड्यूल-आधारित)।

## Routes Router (फ़ाइल-आधारित)

**Routes Router** आपके एप्लिकेशन के URL को स्वचालित रूप से परिभाषित करने के लिए `src/routes/` निर्देशिका की भौतिक संरचना का उपयोग करता है।

- **टोपोलॉजी-ड्रिवेन**: फोल्डर और फ़ाइलें सीधे URL के अनुरूप होती हैं।
- **.nexy, .mdx, .py के लिए समर्थन**: `src/routes/` निर्देशिका की प्रत्येक फ़ाइल एक रूट है।
- **डायनेमिक रूटिंग**: डायनेमिक सेगमेंट के लिए वर्गाकार कोष्ठक `[]` का उपयोग करें (उदाहरण: `src/routes/[id].nexy` -> `/{id}`)।
- **SEO-अनुकूल**: प्रत्येक रूट सर्वर पर रेंडर किया जाता है (SSR)।

| फ़ाइल | URL |
| --- | --- |
| `src/routes/index.nexy` | `/` |
| `src/routes/docs/intro.mdx` | `/docs/intro` |
| `src/routes/api/health.py` | `/api/health` |

## Modular Router (मॉड्यूल-आधारित)

**Modular Router** (`src/app/` में स्थित) जटिल API और उद्यम-स्तरीय अनुप्रयोगों के लिए डिज़ाइन किया गया है। यह स्पष्ट रूट घोषणाओं के लिए क्लास डेकोरेटर्स का उपयोग करता है।

- **मॉड्यूलरिटी**: अपने एप्लिकेशन को `Modules`, `Controllers` और `Providers` में व्यवस्थित करें।
- **निर्भरता इंजेक्शन**: अपनी सेवाओं और संसाधनों को प्रबंधित करने के लिए Nexy के DI सिस्टम का उपयोग करें।
- **स्पष्ट घोषणाएँ**: `@Controller`, `@Get`, `@Post`, आदि जैसे डेकोरेटर्स के साथ अपने रूट परिभाषित करें।
- **परीक्षण क्षमता**: अपने कंट्रोलर और सेवाओं का आसानी से और अलग से परीक्षण करें।

```python
@Controller("/api/v1/users")
class UserController:
    def __init__(self, user_service: UserService):
        self.service = user_service

    @Get("/{id}")
    async def get_profile(self, id: int):
        return await self.service.get_user(id)
```

## हाइब्रिड रूटिंग

Nexy की शक्ति इन दोनों रणनीतियों को संयोजित करने की इसकी क्षमता में निहित है:
1.  अपनी माइक्रोसर्विसेज, API और व्यावसायिक लॉजिक के लिए **Modular Router** का उपयोग करें।
2.  अपने यूजर इंटरफेस, सामग्री और ब्लॉग के लिए **Routes Router** का उपयोग करें।

यह पृथक्करण फ़ाइल-आधारित विकास की गति को बनाए रखते हुए एक स्वच्छ वास्तुशिल्प डिजाइन की अनुमति देता है।

---

**सलाह**: सार्वजनिक पृष्ठों के लिए **Routes Router** और आंतरिक सेवाओं और API के लिए **Modular Router** का उपयोग करें।
