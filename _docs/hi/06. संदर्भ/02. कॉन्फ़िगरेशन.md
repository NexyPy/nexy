# कॉन्फ़िगरेशन (`nexyconfig.py`)

Nexy को न्यूनतम कॉन्फ़िगरेशन (कन्वेंशन ओवर कॉन्फ़िगरेशन) के साथ काम करने के लिए डिज़ाइन किया गया है। हालांकि, जटिल परियोजनाओं के लिए, आप अपने प्रोजेक्ट के रूट में `nexyconfig.py` फ़ाइल के माध्यम से फ़्रेमवर्क के व्यवहार को अनुकूलित कर सकते हैं।

## 1. बुनियादी संरचना

फ़ाइल को `NexyConfigModel` से विरासत में मिली एक `NexyConfig` क्लास परिभाषित करनी होगी।

```python
from nexy.core.models import NexyConfigModel

class NexyConfig(NexyConfigModel):
    useTitle = "मेरा Nexy एप्लिकेशन"
    useVite = True  # Vite ऑर्केस्ट्रेशन सक्रिय करें
```

## 2. सर्वर विकल्प

- **`usePort`** (`int`, डिफ़ॉल्ट: `3000`): वह पोर्ट जिस पर विकास और उत्पादन सर्वर सुनेंगे।
- **`useHost`** (`str`, डिफ़ॉल्ट: `"0.0.0.0"`): सर्वर सुनने वाला होस्ट।
- **`useDocs`** (`bool`, डिफ़ॉल्ट: `True`): FastAPI के Swagger UI इंटरफ़ेस को सक्रिय या निष्क्रिय करता है।
- **`useDocsUrl`** (`str`, डिफ़ॉल्ट: `"/docs"`): Swagger दस्तावेज़ीकरण के लिए एक्सेस URL।
- **`useRedocsUrl`** (`str`, डिफ़ॉल्ट: `"/redocs"`): ReDoc दस्तावेज़ीकरण के लिए एक्सेस URL।

## 3. फ्रंटएंड ऑर्केस्ट्रेशन (Vite)

- **`useVite`** (`bool`, डिफ़ॉल्ट: `False`): Vite एकीकरण सक्रिय करें। यदि आप React, Vue, Svelte या Solid.js घटकों का उपयोग करते हैं तो अपरिहार्य है।
- **`useViteDevUrl`** (`str`, वैकल्पिक): Vite विकास सर्वर के लिए कस्टम URL (उदाहरण: `http://localhost:5173`)।
- **`useAliases`** (`dict`, डिफ़ॉल्ट: `{}`): आपके आयात के लिए पथ उपनाम परिभाषित करता है (उदाहरण: `{"@": "src"}`)।

## 4. एक्सटेंशन और फ़्रेमवर्क (useFF)

Nexy `useFF` (Frontend Frameworks) सूची के माध्यम से अपनी क्षमताओं के विस्तार की अनुमति देता है।

```python
from nexy.core.models import FFModel

class NexyConfig(NexyConfigModel):
    useFF = [
        FFModel(
            name="react",
            render="render_react", # आंतरिक रेंडरिंग फ़ंक्शन
            extension=[".tsx", ".jsx"]
        )
    ]
```

## 5. सामग्री और मार्कडाउन

- **`useMarkdownExtensions`** (`list[str]`, डिफ़ॉल्ट: `[]`): `.mdx` फ़ाइलों के रेंडरिंग के लिए लोड किए जाने वाले Python-Markdown एक्सटेंशन की सूची (उदाहरण: `["extra", "codehilite"]`)।

## 6. रूटिंग और अपवर्जन

- **`excludeDirs`** (`list[str]`, डिफ़ॉल्ट: `[]`): स्वचालित रूट खोज (फ़ाइल-आधारित रूटिंग) के दौरान अनदेखा की जाने वाली निर्देशिकाओं की सूची।
- **`useRouter`** (`APIRouter`, वैकल्पिक): स्टार्टअप पर कस्टम `APIRouter` इंस्टेंस इंजेक्ट करने की अनुमति देता है।

---

**सलाह**: अधिकांश विकल्पों में स्वस्थ डिफ़ॉल्ट मान होते हैं। केवल वही संशोधित करें जो आपके विशिष्ट उपयोग के मामले के लिए आवश्यक है।
