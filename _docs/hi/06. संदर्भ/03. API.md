# API संदर्भ

यह पृष्ठ Nexy में उपलब्ध सभी सार्वजनिक API, डेकोरेटर्स और उपयोगिताओं को सूचीबद्ध करता है।

## डेकोरेटर्स (`nexy.decorators`)

Nexy आपके एप्लिकेशन को संरचित करने और निर्भरताओं को प्रबंधित करने के लिए क्लास डेकोरेटर्स का उपयोग करता है।

### `@Controller(prefix: str = "", tags: List[str] = None)`
एक रूटिंग कंट्रोलर को परिभाषित करता है। HTTP वर्ब (`get`, `post`, आदि) के नाम पर रखे गए क्लास मेथड या WebSockets के लिए `socket` मेथड स्वचालित रूप से प्रवेश बिंदु (endpoints) बन जाते हैं।

### `@Module(prefix: str = "")`
एक मॉड्यूल को परिभाषित करता है। मॉड्यूल कंट्रोलर, प्रदाताओं (providers) और अन्य मॉड्यूल को समूहित करते हैं।
- `controllers`: कंट्रोलर क्लास की सूची।
- `providers`: इंजेक्ट करने योग्य सेवाओं की सूची।
- `imports`: शामिल किए जाने वाले अन्य मॉड्यूल की सूची।

### `@Injectable()`
एक क्लास को ऐसी सेवा के रूप में चिह्नित करता है जिसे कंस्ट्रक्टर के माध्यम से कंट्रोलर या अन्य सेवाओं में इंजेक्ट किया जा सकता है।

### `UseRoute(...)`
एक विशिष्ट रूट के लिए OpenAPI मेटाडेटा (सारांश, विवरण, टैग, आदि) को अनुकूलित करता है।

### `UseResponse(...)`
HTTP प्रतिक्रिया मेटाडेटा (स्थिति कोड, प्रतिक्रिया मॉडल, आदि) को अनुकूलित करता है।

### `UseGuard(*guards)`
प्रमाणीकरण या प्राधिकरण के लिए एक विशिष्ट कंट्रोलर या रूट में एक या अधिक "गार्ड" जोड़ता है।

### `UseMiddleware(*middlewares)`
एक विशिष्ट कंट्रोलर या रूट में एक या अधिक मिडलवेयर जोड़ता है।

## मुख्य घटक और उपयोगिताएँ

### `Vite()`
Vite के लिए आवश्यक स्क्रिप्ट इंजेक्ट करता है (विकास में HMR, उत्पादन में एसेट लिंक)। आमतौर पर लेआउट में स्वचालित रूप से प्रबंधित किया जाता है।

### `Template()`
Jinja2 या मार्कडाउन टेम्प्लेट के मैन्युअल रेंडरिंग के लिए उपयोगिता क्लास।
- `render(path, context)`: रेंडर की गई HTML स्ट्रिंग लौटाता है।

### `Import(path: str)`
क्लाइंट-साइड घटक (React, Vue, आदि) को मैन्युअल रूप से आयात करता है। `.nexy` फ़ाइलों में, इसे कंपाइलर द्वारा स्वचालित रूप से नियंत्रित किया जाता है।

### `Link()`
सुसंगत एप्लिकेशन लिंक उत्पन्न करने के लिए उपयोगिता फ़ंक्शन।

### `t(key: str, default: str = None, locale: str = None)`
अंतर्राष्ट्रीयकरण फ़ंक्शन। वर्तमान या निर्दिष्ट लोकेल के आधार पर अनुवाद स्ट्रिंग प्राप्त करता है।

## एप्लिकेशन ऑब्जेक्ट

### `app`
Nexy द्वारा उपयोग किया जाने वाला नेटिव **FastAPI** इंस्टेंस। आप इसका उपयोग ग्लोबल मिडलवेयर पंजीकृत करने, कस्टम अपवाद हैंडलर बनाने या FastAPI की उन्नत सुविधाओं तक पहुँचने के लिए कर सकते हैं।

```python
from nexy import app

@app.on_event("startup")
async def startup_event():
    print("Nexy एप्लिकेशन शुरू हो गया है!")
```

## मल्टीमीडिया और फॉर्म

Nexy सामान्य वेब तत्वों को प्रबंधित करने के लिए समर्पित क्लास प्रदान करता है। हालांकि वे वर्तमान में संरचित रैपर के रूप में काम करते हैं, वे भविष्य के अनुकूलन के लिए आधार बनाते हैं।

- **`Audio`**: ऑडियो प्लेबैक के लिए उपयोगिताएँ।
- **`Video`**: वीडियो एकीकरण के लिए उपयोगिताएँ।
- **`Image`**: अनुकूलित इमेज रेंडरिंग के लिए उपयोगिताएँ।
- **`Form`**: संरचित फॉर्म प्रबंधन के लिए उपयोगिताएँ।
- **`Script`**: बाहरी या इनलाइन स्क्रिप्ट प्रबंधन के लिए उपयोगिताएँ।

---

## हुक्स (`nexy.hooks`)

- **`useNexyRender(path)`**: कस्टम रेंडरिंग लॉजिक के लिए हुक।
- **`useWebHook(path)`**: वेबहुक प्रबंधन के लिए हुक।
- **`UseStorage()`**: स्टोरेज सिस्टम के साथ बातचीत के लिए हुक।

---

## अंतर्निहित प्रकार

### `prop[T]`
Pydantic सत्यापन के साथ घटक गुणों को घोषित करने के लिए `.nexy` फ़ाइल के पायथन ब्लॉक में उपयोग किया जाता है।

```python
user : prop[UserSchema]
```

---

**सलाह**: भविष्य के अपडेट के साथ संगतता सुनिश्चित करने के लिए इन उपयोगिताओं को हमेशा सीधे `nexy` या `nexy.decorators` से आयात करें।
