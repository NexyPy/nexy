# क्विक स्टार्ट: API से फुलस्टैक तक सिर्फ 5 मिनट में

यह गाइड उन डेवलपर्स के लिए डिज़ाइन की गई है जो Nexy के लचीलेपन को जल्दी से समझना चाहते हैं, एक मजबूत बैकएंड से शुरू करके एक हाइब्रिड इंटरफ़ेस पर समाप्त होते हैं।

## 1. स्थापना

```bash
pip install nexy
```

## 2. "प्योर बैकएंड" प्रोजेक्ट शुरू करना

फिलहाल बिना फ्रंटएंड इंटरफ़ेस के एक प्रोजेक्ट बनाना शुरू करते हैं।

```bash
nx init my-project --no-frontend
cd my-project
```

## 3. एक मॉड्यूलर API बनाना

`src/app/main.py` खोलें और एक साधारण कंट्रोलर परिभाषित करें:

```python
from nexy.decorators import Controller

@Controller("/api")
class HelloController:
    # मेथड का नाम HTTP वर्ब (get, post, आदि) को परिभाषित करता है
    async def get(self):
        return {"status": "Nexy is running", "mode": "Pure Backend"}
```

सर्वर शुरू करें:
```bash
nx dev
```
`http://localhost:8000/api` पर जाएँ। अब आपके पास एक सुपर-फास्ट और मॉड्यूलर FastAPI बैकएंड है।

## 4. फुलस्टैक की ओर माइग्रेशन ("वाह" प्रभाव)

अब, अपने बैकएंड को संशोधित किए बिना एक यूजर इंटरफेस जोड़ते हैं। बस एक `src/routes/index.nexy` फ़ाइल बनाएँ:

```html
---
# एकीकृत पायथन लॉजिक
from datetime import datetime
from "@/components/Counter.tsx" import Counter

now = datetime.now().strftime("%H:%M:%S")
---
<div class="flex flex-col items-center justify-center min-h-screen bg-gray-900 text-white">
    <h1 class="text-5xl font-extrabold text-blue-400">Nexy Fullstack</h1>
    <p class="mt-4 text-xl">सर्वर कहता है कि समय है: <span class="font-mono text-yellow-400">{{ now }}</span></p>
    
    <!-- एक मानक आयात के माध्यम से React घटक का एकीकरण -->
    <div class="mt-8 p-6 bg-gray-800 rounded-xl shadow-2xl">
        <h2 class="text-lg mb-4">क्लाइंट अन्तरक्रियाशीलता (React)</h2>
        <Counter />
    </div>
</div>
```

## 5. ऑर्केस्ट्रेशन क्रिया में

अब पूर्ण विकास सर्वर शुरू करें:
```bash
nx dev
```
Nexy अब:
1.  आपका पायथन बैकएंड (FastAPI) शुरू करेगा।
2.  Vite ऑर्केस्ट्रेटर शुरू करेगा (यदि nexyconfig.py में सक्रिय है)।
3.  `http://localhost:8000` पर आपके पृष्ठ को सेवा देने के लिए दोनों को मर्ज करेगा।

---

## यह प्रवाह अलग क्यों है?

- **प्रगतिशीलता**: आप फुलस्टैक करने के लिए मजबूर नहीं हैं। Nexy एक शुद्ध बैकएंड या एक SSR मोनोलिथ हो सकता है।
- **ज़ीरो ग्लू कोड**: आपके प्रारंभिक डेटा के लिए कोई जटिल `fetch()` कॉल नहीं है; वे पहले से ही आपके पायथन ब्लॉक में `props` के माध्यम से मौजूद हैं, और टाइप किए गए हैं।
- **रेंडरिंग गति**: HTML सर्वर पर जेनरेट किया जाता है, Vite केवल समृद्ध क्लाइंट-साइड अन्तरक्रियाशीलता को संभालता है।
