# अंतर्राष्ट्रीयकरण (i18n)

Nexy आपके अनुप्रयोगों को वैश्विक दर्शकों के लिए सुलभ बनाने के लिए एक सरल और उच्च-प्रदर्शन अंतर्राष्ट्रीयकरण प्रणाली को एकीकृत करता है।

## 1. फ़ाइल संरचना

अनुवाद फ़ाइलें JSON फ़ाइलों के रूप में `locales/` निर्देशिका (या आपके कॉन्फ़िगरेशन में परिभाषित) में संग्रहीत की जाती हैं।

```text
my-project/
├── locales/
│   ├── fr.json
│   └── en.json
└── src/
    └── routes/
        └── index.nexy
```

### `fr.json` फ़ाइल का उदाहरण
```json
{
  "welcome": "Nexy में आपका स्वागत है",
  "description": "पायथन फुलस्टैक फ़्रेमवर्क"
}
```

## 2. .nexy फ़ाइलों में उपयोग

आप सीधे अपने टेम्प्लेट या पायथन लॉजिक में `t()` (translate) फ़ंक्शन का उपयोग कर सकते हैं।

```html
---
from nexy import t
---
<section>
    <h1>{{ t("welcome") }}</h1>
    <p>{{ t("description") }}</p>
</section>
```

## 3. भाषा पहचान

Nexy स्वचालित रूप से इसके माध्यम से उपयोगकर्ता की भाषा का पता लगाता है:
1.  **URL पैरामीटर**: `?lang=fr`
2.  **HTTP हेडर**: `Accept-Language`
3.  **कुकी**: `nexy_locale`

## 4. गतिशील अनुवाद (चर)

आप अपनी स्ट्रिंग्स में चर पास कर सकते हैं।

### JSON फ़ाइल
```json
{
  "greeting": "नमस्ते {name} !"
}
```

### उपयोग
```python
{{ t("greeting", name="Alice") }}
```

## 5. कॉन्फ़िगरेशन

आप अपने `nexyconfig.py` में डिफ़ॉल्ट भाषा परिभाषित कर सकते हैं।

```python
from nexy.core.models import NexyConfigModel

class NexyConfig(NexyConfigModel):
    useLocaleDefault = "fr"
    useLocales = ["fr", "en"]
```

---

**सलाह**: अपने प्रोजेक्ट की शुरुआत से ही अंतर्राष्ट्रीयकरण का उपयोग करें ताकि भविष्य में आपके एप्लिकेशन का विस्तार करना आसान हो सके।
