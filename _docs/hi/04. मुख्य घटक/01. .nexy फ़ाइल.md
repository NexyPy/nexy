# मुख्य घटक: .nexy और .mdx

Nexy इंटरफेस बनाने के लिए दो मुख्य संरचना इकाइयाँ प्रदान करता है: **.nexy** फ़ाइलें और **.mdx** फ़ाइलें।

## .nexy फ़ाइल

एक `.nexy` फ़ाइल एक **सर्वर घटक** है जो पायथन लॉजिक और HTML टेम्प्लेट (Jinja2) को जोड़ती है।

### संरचना

एक `.nexy` फ़ाइल को दो खंडों में विभाजित किया गया है:

1.  **लॉजिक ब्लॉक (पायथन)**: `---` मार्करों के बीच संलग्न।
2.  **प्रस्तुति ब्लॉक (टेम्प्लेट)**: Jinja2 सिंटैक्स के साथ संवर्धित HTML।

```python
---
# पायथन लॉजिक (सर्वर-साइड)
from pydantic import BaseModel

class UserProfile(BaseModel):
    id: int
    username: str

user : prop[UserProfile]
theme : prop[str] = "dark"

# क्लाइंट घटकों का आयात
from "@/components/Avatar.tsx" import Avatar  # क्लाइंट घटक
---
```

```html
<!-- HTML/Jinja2 टेम्प्लेट -->
<section class="profile-container {{ theme }}">
    <header>
        <Avatar src={user.avatar_url} />
        <h1>{{ user.username }}</h1>
    </header>
    
    <main>
        <p>उपयोगकर्ता ID: {{ user.id }}</p>
    </main>
</section>
```

### मुख्य विशेषताएं

- **Pydantic Props**: Pydantic स्कीमा के साथ अपने गुणों को परिभाषित और सत्यापित करें।
- **निर्भरता इंजेक्शन**: पायथन ब्लॉक में सेवाओं को इंजेक्ट करने के लिए Nexy के DI सिस्टम का उपयोग करें।
- **सर्वर रेंडरिंग**: शुद्ध HTML उत्पन्न करता है जो क्लाइंट को परोसा जाता है।

## MDX समर्थन

Nexy मूल रूप से **MDX** का समर्थन करता है, जो आपको एम्बेडेड घटकों के साथ मार्कडाउन लिखने की अनुमति देता है।

- **सामग्री-प्रथम**: ब्लॉग, दस्तावेज़ीकरण और सामग्री-प्रधान पृष्ठों के लिए आदर्श।
- **घटक-संचालित**: अपने मार्कडाउन के भीतर React, Vue, Svelte या Solid.js घटकों को आयात और उपयोग करें।
- **SEO-अनुकूल**: सर्वर पर HTML में रेंडर किया गया, बिल्कुल `.nexy` फ़ाइलों की तरह।

### MDX उदाहरण

```mdx
# मेरा पहला ब्लॉग लेख

मेरे ब्लॉग पर आपका स्वागत है! यहाँ एक React घटक है:

import Counter from '@/components/Counter.tsx'

<Counter initial={10} />

यह **markdown** में एक पैराग्राफ है।
```

---

**सलाह**: एप्लिकेशन लॉजिक और जटिल लेआउट के लिए `.nexy` का उपयोग करें, और दस्तावेज़ीकरण और ब्लॉग सामग्री के लिए `.mdx` का उपयोग करें।
